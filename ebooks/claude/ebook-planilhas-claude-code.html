Introdução

Por que a planilha parou de ser suficiente

A planilha foi uma das invenções mais úteis da história do trabalho de escritório. Em 1979, quando o VisiCalc apareceu no Apple II, ele substituiu horas de cálculo manual por segundos de processamento automático. Décadas depois, o Excel e o Google Sheets ainda dominam o dia a dia de empresas de todos os tamanhos. O problema não é a ferramenta em si: é o que as empresas passaram a exigir dela.

Uma planilha foi projetada para organizar dados em linhas e colunas e fazer cálculos sobre eles. Ela não foi projetada para ser um sistema de controle de estoque, um CRM, um sistema de folha de pagamento ou um painel de indicadores em tempo real. Quando uma empresa usa a planilha para tudo isso ao mesmo tempo, surgem os problemas conhecidos: arquivos que travam, fórmulas que quebram sem aviso, dados duplicados em abas diferentes, versões conflitantes circulando por e-mail e erros que só aparecem semanas depois, quando o prejuízo já aconteceu.

O Claude Code muda essa equação de uma forma que não exige que você aprenda programação. Ele é uma ferramenta de inteligência artificial que entende o que você descreve em português, gera o código necessário para resolver o problema e explica o que cada parte faz. Você não precisa entender o código. Você precisa entender o problema, descrevê-lo com clareza e saber onde rodar o resultado.

O que este guia cobre

Este guia ensina como usar o Claude Code para três coisas distintas: melhorar planilhas que já existem, automatizar tarefas que hoje são feitas manualmente e, quando fizer sentido, substituir a planilha por uma aplicação web simples que roda no seu próprio computador.

Os cenários cobertos são os mais comuns no ambiente empresarial: financeiro, vendas, estoque, recursos humanos e operações. Para cada um, você vai encontrar uma descrição do que o Claude Code consegue fazer, como pedir isso a ele e quando a planilha chegou no limite e precisa ser substituída por algo mais robusto.

O fluxo técnico é sempre o mesmo: você abre o Claude Code, descreve o problema ou envia a planilha, ele gera um script em Python, você abre o Google Colab (ensinado do zero no Capítulo 2), cola o script, roda e obtém o resultado. Nenhuma instalação de Python, nenhuma configuração de ambiente, nenhum conhecimento técnico prévio além do básico de usar o Claude Code.

O que está fora do escopo

Este guia dá direcionamento, não tutoriais exaustivos para cada cenário possível. Cada empresa tem suas particularidades, e o objetivo aqui é que você saia sabendo o que pedir ao Claude Code em cada situação, não que você receba um script pronto para copiar sem pensar. Os exemplos de instrução mostrados ao longo do texto são pontos de partida: você vai adaptá-los à sua realidade.

O pré-requisito é simples: ter o Claude Code instalado e saber o básico de como usá-lo, ou seja, abrir o terminal, digitar uma instrução e ler a resposta. Se você ainda não chegou nesse ponto, instale o Claude Code seguindo a documentação oficial antes de continuar.

Como o Claude Code lida com planilhas

Antes de pedir qualquer automação, vale entender o que o Claude Code consegue enxergar em uma planilha e como descrever o problema de forma que ele entenda sem ambiguidade.

Como enviar uma planilha para análise

O Claude Code aceita arquivos diretamente. Você pode arrastar o arquivo para a janela do terminal onde o Claude Code está rodando, ou usar o comando de upload disponível na interface. Formatos aceitos incluem .xlsx, .xls e .csv. Arquivos do Google Sheets precisam ser exportados primeiro: no Google Sheets, vá em Arquivo, depois Fazer download, e escolha Microsoft Excel ou CSV.

Quando você envia um arquivo, o Claude Code lê a estrutura: quantas abas existem, quais são os nomes das colunas, quantas linhas de dados há, quais células têm fórmulas e quais têm valores fixos. Ele não executa o arquivo, apenas lê o conteúdo. Por isso, planilhas com macros complexas do VBA não têm as macros analisadas, apenas os dados e a estrutura.

O que o Claude Code enxerga em uma planilha bagunçada

Uma planilha bagunçada costuma ter alguns padrões reconhecíveis: cabeçalhos misturados com dados, células mescladas que dificultam a leitura por código, colunas com tipos de dados inconsistentes (datas escritas como texto, valores com símbolo de moeda embutido), linhas em branco no meio dos dados e fórmulas que referenciam células de outras abas sem documentação.

O Claude Code identifica esses padrões quando você pede um diagnóstico. Ele não vai simplesmente corrigir tudo de uma vez sem avisar: se você pedir um diagnóstico antes de qualquer ação, ele vai listar o que encontrou e perguntar como você quer proceder. Esse passo é importante porque algumas "bagunças" são intencionais, como células mescladas usadas para formatação visual em relatórios impressos.

Como descrever o problema com clareza

A qualidade da instrução determina a qualidade do resultado. Instruções vagas geram scripts que resolvem o problema errado. Instruções específicas geram scripts que resolvem exatamente o que você precisa.

Uma instrução vaga seria: "arruma minha planilha de vendas". Uma instrução específica seria: "tenho uma planilha de vendas com as colunas Data, Vendedor, Produto, Quantidade e Valor. A coluna Data tem datas em formatos diferentes: algumas no formato DD/MM/AAAA e outras no formato MM-DD-AAAA. Quero que todas fiquem no formato DD/MM/AAAA e que a planilha seja ordenada por data, da mais antiga para a mais recente."

A diferença é que a instrução específica diz: o que existe, onde está o problema e qual é o resultado esperado. Quando você inclui esses três elementos, o Claude Code gera um script que faz exatamente o que você pediu.

O conceito de diagnóstico: antes de otimizar, entender o que está errado

Antes de pedir qualquer transformação, peça um diagnóstico. O diagnóstico é uma análise sem modificações: o Claude Code lê a planilha e descreve o que encontrou, sem alterar nada. Isso serve para dois propósitos: você confirma que ele entendeu a estrutura corretamente, e você descobre problemas que não sabia que existiam.

É comum que um diagnóstico revele, por exemplo, que uma coluna de valores tem 12% das células com texto em vez de número, o que quebraria qualquer cálculo automático. Saber disso antes de pedir a automação evita que o script gere resultados errados silenciosamente.

Instrução de diagnóstico para o Claude Code

Analise o arquivo "vendas_2025.xlsx" que estou enviando e me dê um diagnóstico completo da estrutura. Liste: quantas abas existem e o que cada uma contém, quais são as colunas de cada aba, se há células mescladas, se há linhas ou colunas em branco no meio dos dados, se há inconsistências de tipo nas colunas (por exemplo, datas escritas como texto ou valores com letras misturadas), e se há fórmulas que podem estar quebradas. Não modifique nada ainda. Apenas descreva o que encontrou.

Como pedir sugestões antes de qualquer mudança

Depois do diagnóstico, o próximo passo é pedir sugestões. Você não precisa saber o que precisa ser feito: o Claude Code pode propor um plano de ação com base no que encontrou. Você aprova, rejeita ou ajusta cada item antes de qualquer script ser gerado.

Instrução para sugestões de melhoria

Com base no diagnóstico que você fez da planilha "vendas_2025.xlsx", liste as melhorias que você recomenda fazer, em ordem de prioridade. Para cada melhoria, explique em uma frase o que ela resolve e qual seria o impacto se não for feita. Não gere nenhum código ainda.

É útil ter clareza sobre quem faz o quê nesse fluxo. O Claude Code entende o problema, gera o código e explica o que ele faz. O Google Colab é o ambiente onde o código roda: você cola o script lá, aperta um botão e o resultado aparece. O seu papel é descrever o problema com precisão, revisar o resultado e decidir se está correto.

Nenhuma dessas três partes substitui as outras. O Claude Code não roda o código por conta própria. O Google Colab não entende o problema. Você não precisa escrever o código. Cada um faz o que sabe fazer melhor.

Google Colab, seu laboratório de automação

Você não precisa instalar Python, configurar ambiente ou entender nada de programação para rodar os scripts gerados pelo Claude Code. O Google Colab faz tudo isso por você, direto no navegador.

O que é o Google Colab

O Google Colab (nome completo: Google Colaboratory) é um serviço gratuito do Google que permite rodar código Python diretamente no navegador, sem instalar nada no computador. Pense nele como um caderno online: você escreve (ou cola) o código em células, aperta um botão e o código roda em servidores do Google. O resultado aparece logo abaixo da célula.

Para usar o Google Colab, você precisa apenas de uma conta Google (a mesma do Gmail) e de um navegador. Não há instalação, não há configuração de versão de Python, não há gerenciamento de bibliotecas na maioria dos casos. As bibliotecas mais comuns para trabalhar com planilhas, como pandas e openpyxl, já estão disponíveis no Colab por padrão.

Por que usar o Google Colab e não instalar Python no computador

Instalar Python localmente envolve escolher a versão correta, configurar variáveis de ambiente, instalar um gerenciador de pacotes e resolver conflitos de dependência. Para quem não é programador, esse processo pode levar horas e resultar em erros difíceis de diagnosticar.

O Google Colab elimina tudo isso. O ambiente já está configurado. Quando um script precisa de uma biblioteca que não está disponível por padrão, basta adicionar uma linha no início do script (o Claude Code faz isso automaticamente) e o Colab instala na hora. Quando você fecha o Colab, o ambiente é descartado. Na próxima vez que abrir, começa limpo.

Como baixar o arquivo gerado

Quando o script gera um arquivo de saída, o arquivo fica salvo no ambiente temporário do Colab. Para baixá-lo, há duas formas. A primeira: o próprio script pode incluir um comando de download automático. O Claude Code costuma incluir isso quando você pede explicitamente. O arquivo vai aparecer na pasta de downloads do seu navegador como qualquer outro download. A segunda: você pode baixar manualmente pelo painel de arquivos. Clique no ícone de pasta no painel esquerdo, localize o arquivo gerado, clique com o botão direito sobre ele e escolha "Fazer download".

Atenção: o ambiente do Colab é temporário

Os arquivos que você faz upload e os arquivos gerados pelo script existem apenas enquanto a sessão do Colab está ativa. Se você fechar o navegador ou a sessão expirar (geralmente após 90 minutos de inatividade), tudo é apagado. Sempre baixe o arquivo gerado antes de fechar.

Erros comuns ao rodar scripts no Colab

Mesmo sem entender Python, você vai encontrar erros eventualmente. A boa notícia é que você não precisa entender o erro para corrigi-lo: basta copiar a mensagem de erro e enviar ao Claude Code pedindo correção.

Os erros mais comuns têm causas simples. O erro FileNotFoundError significa que o script está procurando um arquivo com um nome específico e não encontrou: verifique se o nome do arquivo que você fez upload é exatamente igual ao nome que aparece no script. O erro ModuleNotFoundError significa que uma biblioteca não está instalada: o Claude Code vai adicionar o comando de instalação quando você reportar o erro. O erro KeyError geralmente significa que o script está procurando uma coluna com um nome que não existe na sua planilha: verifique se os nomes das colunas no script correspondem exatamente aos nomes na sua planilha, incluindo maiúsculas, minúsculas e espaços.

Planilhas bagunçadas, o ponto de partida real

A maioria das planilhas empresariais não foi planejada: foi crescendo. Alguém criou uma aba, outra pessoa adicionou colunas, um terceiro copiou dados de outro sistema e colou sem padronizar. O resultado é uma estrutura que funciona na cabeça de quem criou, mas quebra quando qualquer automação tenta processá-la.

Por que a maioria das planilhas empresariais está mal estruturada

Planilhas crescem organicamente. Ninguém senta para projetar uma planilha do zero com regras claras de estrutura. O que acontece na prática é que alguém cria uma aba para controlar algo, outra pessoa adiciona uma coluna para registrar uma informação nova, um terceiro copia dados de um sistema externo e cola sem padronizar o formato. Com o tempo, a planilha acumula camadas de decisões diferentes, feitas por pessoas diferentes, sem documentação.

Isso não é negligência: é a natureza das ferramentas flexíveis. A planilha aceita qualquer coisa em qualquer célula, o que é sua maior vantagem e seu maior problema ao mesmo tempo. Quando você precisa automatizar algo, essa flexibilidade vira obstáculo.

Os problemas mais comuns

Dados misturados com formatação. Células que contêm tanto o valor quanto a unidade de medida (por exemplo, "150 kg" em vez de 150 na célula e "kg" no cabeçalho da coluna). Isso impede qualquer cálculo automático.

Formatação inconsistente. Datas em formatos diferentes na mesma coluna, valores com e sem símbolo de moeda, nomes de produtos com grafias diferentes ("Produto A", "produto a", "PRODUTO A" tratados como itens distintos).

Fórmulas quebradas. Referências a células que foram movidas ou deletadas, resultando em erros como #REF! ou #VALOR! que se propagam silenciosamente por outras fórmulas.

Abas duplicadas. Versões diferentes do mesmo dado em abas diferentes, sem clareza sobre qual é a fonte de verdade.

Células mescladas. Usadas para deixar o relatório bonito visualmente, mas que impedem que o código leia os dados corretamente, porque a mesclagem quebra a estrutura de linhas e colunas.

Linhas de totais no meio dos dados. Subtotais inseridos entre as linhas de detalhe, misturando dados brutos com dados calculados na mesma coluna.

Como pedir ao Claude Code para limpar e reorganizar sem perder dados

A instrução de limpeza precisa ser específica sobre o que preservar e o que transformar. O Claude Code não vai apagar dados sem que você peça explicitamente. Mas é importante deixar claro o que você quer como resultado final.

Instrução de limpeza de planilha

Tenho uma planilha "estoque.xlsx" com uma aba chamada "Movimentações". Os problemas que identifiquei são: (1) a coluna "Quantidade" tem valores como "50 un", "30 un" e "20 peças" misturando número e unidade na mesma célula; (2) a coluna "Data" tem datas em três formatos diferentes: DD/MM/AAAA, MM/DD/AAAA e por extenso como "15 de março de 2025"; (3) há linhas em branco a cada 10 linhas que parecem ser separadores visuais. Gere um script Python que: remova as linhas em branco, padronize a coluna Data para DD/MM/AAAA, separe a coluna Quantidade em duas colunas (Quantidade_Numero e Unidade), e salve o resultado em um novo arquivo chamado "estoque_limpo.xlsx" sem modificar o arquivo original.

O que uma planilha bem estruturada parece

Uma planilha bem estruturada para processamento automático segue algumas regras simples. A primeira linha é sempre o cabeçalho, com um nome único por coluna. Cada linha seguinte é um registro de dado. Não há linhas em branco no meio dos dados. Não há células mescladas. Cada coluna tem um único tipo de dado: datas são sempre datas, números são sempre números, texto é sempre texto. Não há subtotais misturados com os dados brutos: se você precisa de totais, eles ficam em uma aba separada ou são calculados pelo script.

Essa estrutura é chamada de "tabela plana" ou, em inglês, flat table. É o formato que qualquer ferramenta de análise de dados espera receber. Quando sua planilha está nesse formato, qualquer automação funciona sem surpresas.

Como validar o resultado sem entender o código

Depois de rodar o script de limpeza, você precisa verificar se o resultado está correto. Você não precisa entender o código para fazer isso: basta abrir o arquivo gerado e checar visualmente os pontos que você pediu para corrigir. Se você pediu para padronizar datas, abra a coluna de datas e verifique se todas estão no mesmo formato. Se você pediu para remover linhas em branco, role a planilha e veja se ainda há linhas vazias no meio.

Além da verificação visual, peça ao Claude Code para gerar um relatório de validação junto com o script de limpeza. Esse relatório mostra quantas linhas foram processadas, quantas transformações foram feitas e se houve algum dado que não pôde ser convertido.

Quando a planilha está tão bagunçada que o melhor caminho é reconstruir do zero

Às vezes a limpeza custa mais do que a reconstrução. Isso acontece quando a estrutura original é tão inconsistente que qualquer tentativa de padronização automática vai gerar mais erros do que corrigir. Os sinais são: mais de 30% das células com dados em formato incorreto, múltiplas abas com dados sobrepostos sem critério claro, fórmulas que referenciam outras fórmulas em cadeia de mais de cinco níveis.

Nesse caso, o caminho mais eficiente é pedir ao Claude Code para criar uma planilha nova com a estrutura correta e, em seguida, gerar um script que extrai os dados aproveitáveis da planilha antiga e os insere na nova estrutura. Você perde a formatação visual, mas ganha uma base sólida para trabalhar.

Instrução para reconstrução de planilha

A planilha "controle_financeiro_antigo.xlsx" está muito bagunçada para limpar diretamente. Com base no diagnóstico que você fez, crie uma nova planilha com estrutura correta para controle financeiro, com as colunas: Data (formato DD/MM/AAAA), Tipo (Receita ou Despesa), Categoria, Descrição, Valor (número sem símbolo de moeda), Centro_de_Custo. Depois, gere um script separado que tenta extrair os dados da planilha antiga e os insere na nova estrutura, marcando com "REVISAR" qualquer linha onde a conversão não foi possível com certeza.

Cenários financeiros

O setor financeiro é onde planilhas mal estruturadas causam mais dano. Um erro de fórmula em um fluxo de caixa pode levar a decisões erradas sobre pagamentos e investimentos. O Claude Code não elimina o risco humano de inserir dados errados, mas elimina o risco de fórmulas quebradas e cálculos manuais incorretos.

Fluxo de caixa

O fluxo de caixa é o controle de todas as entradas e saídas de dinheiro da empresa em um período. Na planilha, o problema mais comum é que os dados ficam espalhados: receitas em uma aba, despesas em outra, transferências em uma terceira, e o resumo mensal é feito manualmente copiando e colando valores. Quando um lançamento é esquecido ou duplicado, o saldo fica errado e ninguém percebe até o extrato bancário chegar.

O Claude Code pode consolidar todas as abas em uma única tabela de lançamentos, calcular o saldo acumulado dia a dia, identificar os meses com maior concentração de saídas e gerar um resumo mensal automático com totais por categoria.

Instrução para fluxo de caixa

Tenho uma planilha "fluxo_caixa.xlsx" com três abas: "Receitas" (colunas: Data, Descrição, Valor, Categoria), "Despesas" (colunas: Data, Descrição, Valor, Categoria, Fornecedor) e "Transferências" (colunas: Data, Origem, Destino, Valor). Gere um script Python que: (1) consolide as três abas em uma única tabela com as colunas Data, Tipo (Receita/Despesa/Transferência), Descrição, Valor, Categoria; (2) ordene por data; (3) calcule uma coluna de Saldo_Acumulado; (4) gere uma segunda aba chamada "Resumo_Mensal" com o total de receitas, total de despesas e saldo líquido por mês; (5) salve tudo em "fluxo_consolidado.xlsx".

Quando substituir por aplicação web: quando mais de duas pessoas precisam lançar dados ao mesmo tempo, ou quando o volume de lançamentos diários ultrapassa 50 registros e a planilha começa a ficar lenta.

Contas a pagar e receber

O controle de contas a pagar e receber em planilha tem um problema estrutural: a planilha não avisa quando um vencimento está chegando. Você precisa abrir o arquivo, olhar as datas e identificar manualmente o que está vencendo. Com o Claude Code, você pode gerar um relatório de vencimentos automaticamente, com destaque para o que já venceu e o que vence nos próximos sete dias.

Consolidação de múltiplas planilhas

Consolidar dados de várias planilhas em um único relatório é uma das tarefas mais comuns e mais tediosas do trabalho com planilhas. Cada filial envia sua planilha mensal, cada departamento tem seu arquivo, cada vendedor registra em um arquivo separado. Juntar tudo manualmente leva horas e introduz erros de cópia.

Instrução para consolidação de planilhas

Tenho uma pasta com 12 arquivos Excel, um por mês do ano, todos com o mesmo formato: colunas Data, Filial, Produto, Quantidade, Valor. Os arquivos se chamam "vendas_jan.xlsx", "vendas_fev.xlsx", etc. Gere um script Python que: (1) leia todos os arquivos da pasta que seguem o padrão "vendas_*.xlsx"; (2) consolide todos em uma única tabela adicionando uma coluna "Arquivo_Origem" para rastreabilidade; (3) verifique se todos os arquivos têm as mesmas colunas e avise se algum tiver estrutura diferente; (4) gere um resumo por Filial e por Produto com totais anuais; (5) salve em "consolidado_anual.xlsx" com uma aba de dados brutos e uma aba de resumo.

Quando substituir por aplicação web: quando as filiais ou departamentos precisam inserir os dados diretamente em um sistema central, eliminando o processo de envio e consolidação de arquivos.

Quando substituir a planilha por uma aplicação web

Automatizar uma planilha com scripts resolve muitos problemas, mas não todos. Há um ponto em que a planilha, mesmo bem estruturada e com automações, não consegue mais atender ao que a empresa precisa. Reconhecer esse ponto antes que os problemas se acumulem é o que separa uma decisão planejada de uma crise.

Os sinais de que a planilha chegou no limite

Múltiplos usuários editando ao mesmo tempo. O Google Sheets permite edição simultânea, mas sem controle de acesso por campo: qualquer pessoa pode editar qualquer célula. Quando duas pessoas editam a mesma célula ao mesmo tempo, uma sobrescreve a outra sem aviso. O Excel compartilhado tem problemas ainda maiores de conflito de versão.

Dados perdidos ou sobrescritos sem rastreabilidade. Planilhas não têm histórico de alterações por célula de forma nativa. Quando um dado importante é apagado ou alterado por engano, não há como saber quem fez, quando fez ou qual era o valor anterior.

Fórmulas quebrando com frequência. Quando a planilha tem muitas fórmulas interdependentes e qualquer inserção ou exclusão de linha quebra o cálculo, o custo de manutenção começa a superar o benefício de usar a planilha.

Arquivo pesado demais. Planilhas com mais de 50 mil linhas ou com muitas fórmulas complexas ficam lentas para abrir, salvar e calcular. Isso é um sinal de que o volume de dados cresceu além do que a ferramenta foi projetada para suportar.

Necessidade de acesso por dispositivos móveis. Planilhas não funcionam bem em celulares. Se a equipe precisa registrar ou consultar dados em campo, a planilha vira um obstáculo.

O que uma aplicação web local oferece que a planilha não oferece

Uma aplicação web simples rodando no seu computador (acessada pelo navegador via endereço local, sem precisar de internet) oferece: controle de acesso por usuário, histórico de todas as alterações, formulários que validam os dados antes de salvar, relatórios gerados automaticamente e acesso simultâneo por múltiplos usuários sem conflito.

O padrão técnico usado neste guia é: Next.js para a interface web, SQLite para o banco de dados e execução local via terminal. Você acessa pelo navegador no endereço localhost:3000. Não há servidor externo, não há mensalidade, não há dependência de internet. Os dados ficam no seu computador.

O que esperar do processo

Criar uma aplicação web simples com o Claude Code leva entre 30 minutos e 2 horas, dependendo da complexidade. O Claude Code vai gerar o código, você vai rodar no terminal e testar no navegador. É provável que alguns ajustes sejam necessários: um campo que ficou faltando, um relatório que precisa de um filtro adicional, uma validação que não está funcionando como esperado. Quando a aplicação web também não é a resposta certa

Uma aplicação web local resolve o problema de múltiplos usuários e rastreabilidade, mas cria uma nova responsabilidade: manutenção técnica. Se o computador onde a aplicação roda for formatado, os dados podem ser perdidos se não houver backup. Se o Node.js for atualizado e quebrar a compatibilidade, alguém precisa corrigir.

Para empresas com mais de 20 usuários simultâneos ou com necessidade de acesso remoto por múltiplas localidades, a solução correta é um sistema hospedado em servidor, o que está além do escopo deste guia. Nesse caso, o Claude Code pode ajudar a especificar os requisitos para contratar um sistema adequado ou para desenvolver uma solução mais robusta com um desenvolvedor.

Boas práticas e erros comuns

A maioria dos problemas ao usar o Claude Code com planilhas não vem do Claude Code: vem de instruções imprecisas, arquivos mal organizados ou da ausência de backup antes de qualquer automação. Este capítulo cobre os erros mais comuns e como evitá-los.

Como organizar os arquivos antes de enviar ao Claude Code

Antes de enviar qualquer planilha para análise ou automação, faça uma verificação rápida. Primeiro, certifique-se de que o arquivo tem um nome descritivo que inclua o conteúdo e o período: "vendas_janeiro_2025.xlsx" é melhor do que "planilha1.xlsx". Segundo, verifique se o arquivo abre sem erros no Excel ou Google Sheets. Terceiro, se a planilha tem múltiplas abas, saiba o nome de cada aba e o que ela contém antes de descrever o problema ao Claude Code.

Arquivos com senhas precisam ter a senha removida antes do envio. O Claude Code não consegue abrir arquivos protegidos por senha.

A importância do backup antes de qualquer automação

Antes de rodar qualquer script que modifica uma planilha, faça uma cópia do arquivo original. Renomeie a cópia com o sufixo "_backup_[data]" e guarde em uma pasta separada. Isso leva 30 segundos e pode salvar horas de retrabalho se o script gerar um resultado inesperado.

Uma boa prática é pedir ao Claude Code que o script sempre salve o resultado em um arquivo novo, sem modificar o arquivo original. A instrução "salve o resultado em um novo arquivo sem modificar o arquivo original" deve estar em toda instrução de transformação de planilha.

O arquivo abre sem erros no Excel ou Google Sheets
Sei o nome de todas as abas e o que cada uma contém
Sei os nomes exatos de todas as colunas relevantes

Como descrever um problema sem ambiguidade

A instrução ideal ao Claude Code tem três partes: o que existe (estrutura atual da planilha), o que está errado ou o que você quer fazer, e o que você quer como resultado. Quando uma dessas partes está faltando, o Claude Code vai fazer suposições que podem não corresponder à sua realidade.

Evite instruções abertas como "analise minha planilha e melhore o que precisar". Isso gera um resultado genérico que pode não resolver o seu problema específico. Prefira instruções fechadas: "a planilha tem o problema X, quero que o resultado seja Y, o arquivo se chama Z e tem as colunas A, B e C".

Quando você não sabe exatamente o que está errado, use o diagnóstico primeiro (descrito no Capítulo 1) para que o Claude Code identifique os problemas. Depois, com a lista de problemas em mãos, você pode dar instruções específicas para cada um.

Os erros mais comuns e como evitá-los

Nomes de colunas diferentes entre a instrução e a planilha. Se você diz "a coluna se chama Valor_Total" mas na planilha ela se chama "Valor Total" (com espaço em vez de underscore), o script vai dar erro. Sempre copie os nomes das colunas diretamente da planilha para a instrução.

Não especificar o formato de data esperado. Python lida com datas de forma diferente do Excel. Se você não especificar o formato de saída, o script pode gerar datas no formato americano (MM/DD/AAAA) em vez do brasileiro (DD/MM/AAAA). Sempre inclua o formato de data desejado na instrução.

Esquecer de fazer upload do arquivo no Colab. O script roda, mas dá FileNotFoundError porque o arquivo não foi carregado. Verifique sempre se o arquivo aparece no painel de arquivos do Colab antes de rodar o script.

Pedir tudo de uma vez em uma instrução muito longa. Instruções com muitos requisitos simultâneos aumentam a chance de o script ter erros. Prefira dividir em etapas: primeiro limpe os dados, depois faça os cálculos, depois formate o relatório.

Como iterar: pedir, ver o resultado, corrigir, pedir de novo

O fluxo de trabalho com o Claude Code não é linear. Você pede, roda, vê o resultado, identifica o que não está certo e pede a correção. Cada iteração melhora o resultado. Não tente acertar tudo na primeira instrução: é mais eficiente começar com uma instrução simples, verificar o resultado e adicionar requisitos gradualmente.

Quando o resultado está quase certo mas tem um detalhe errado, descreva apenas o detalhe que precisa mudar. Não repita toda a instrução original: o Claude Code mantém o contexto da conversa e sabe o que foi pedido antes.

Cuidados com dados sensíveis

Não envie ao Claude Code planilhas que contenham: CPF, RG, número de passaporte, dados bancários (número de conta, agência, chave PIX), senhas, informações de saúde, salários nominais ou qualquer dado que possa identificar individualmente uma pessoa física.

Se você precisa automatizar uma planilha que contém esses dados, há duas opções. A primeira: substitua os dados sensíveis por dados fictícios antes de enviar, peça ao Claude Code para gerar o script com base nos dados fictícios e depois aplique o script na planilha real localmente. A segunda: descreva a estrutura da planilha ao Claude Code sem enviar o arquivo, e peça que ele gere o script com base na descrição.
