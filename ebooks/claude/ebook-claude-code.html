Introdução

O Desenvolvimento de Software Mudou. Você Não Precisa Ficar Para Trás.

Por décadas, criar um software exigiu anos de estudo, domínio de linguagens de programação e uma equipe técnica cara. Isso mudou. Não de forma gradual: de forma abrupta.

Durante muito tempo, a barreira entre uma ideia de negócio e o software que a executa foi o código. Você podia ter clareza total sobre o que queria construir, um mercado esperando, recursos disponíveis, e ainda assim precisava de um desenvolvedor para transformar sua visão em realidade. Essa dependência moldou como empresas eram criadas, quanto custavam e quem podia criá-las.

A inteligência artificial mudou essa equação. Não porque os computadores passaram a "pensar" no sentido humano, mas porque modelos de linguagem treinados em bilhões de linhas de código aprenderam a escrever, revisar, depurar e organizar software com uma competência que, em muitos contextos, supera a de desenvolvedores experientes. O que antes levava semanas pode ser feito em horas. O que exigia uma equipe pode ser feito por uma pessoa com as ferramentas certas.

O Claude Code é uma dessas ferramentas. Mas ele não é um chatbot onde você cola uma pergunta e recebe uma resposta. É um agente de desenvolvimento: um programa que lê seu projeto inteiro, entende o que você quer construir, escreve o código, executa testes, corrige erros e itera, tudo dentro do seu computador, com você supervisionando cada passo.

Este ebook foi escrito para pessoas que têm ideias de software mas não sabem programar. Você vai aprender o que é o Claude Code, como instalá-lo, como dar instruções eficazes, como usar recursos avançados como Skills e MCP, como manter seu projeto seguro e como trabalhar de forma sustentável ao longo do tempo. Ao final, você terá o mapa completo para criar sistemas reais, sejam ERPs, plataformas SaaS, automações ou ferramentas internas, sem precisar aprender a programar do zero.

O que é o Claude Code

Claude Code não é um assistente que responde perguntas. É um agente que age: lê arquivos, escreve código, executa comandos e itera até o resultado estar correto.

Definição clara: o que é e o que não é

O Claude Code é uma ferramenta de linha de comando desenvolvida pela Anthropic, a mesma empresa que criou o modelo de inteligência artificial Claude. Ele roda no seu computador, dentro de um terminal, e tem acesso direto aos arquivos do seu projeto. Isso significa que ele não apenas sugere código: ele lê o que já existe, entende a estrutura do projeto, escreve novos arquivos, modifica os existentes e executa comandos no sistema.

O que ele não é: não é um chatbot de perguntas e respostas. Não é um gerador de código que você copia e cola manualmente. Não é uma ferramenta que trabalha em isolamento, sem ver o contexto do seu projeto. E não é autônomo no sentido de que age sem você saber: por padrão, ele pede confirmação antes de executar ações que modificam arquivos ou rodam comandos no sistema.

A diferença entre Claude e Claude Code

Claude é o modelo de linguagem da Anthropic, acessível pelo site claude.ai. Você abre o navegador, digita uma pergunta, recebe uma resposta. É útil para redigir textos, explicar conceitos, resumir documentos. Mas quando você fecha a janela, o Claude não sabe nada sobre o que você estava fazendo antes. Cada conversa começa do zero.

O Claude Code é diferente em três aspectos fundamentais. Primeiro, ele tem acesso ao seu sistema de arquivos: pode ler qualquer arquivo do seu projeto, entender como as peças se conectam e escrever código que se encaixa no que já existe. Segundo, ele pode executar comandos no terminal, como instalar dependências, rodar testes ou iniciar um servidor. Terceiro, ele mantém o contexto do projeto ao longo de uma sessão de trabalho, o que significa que ele "lembra" o que foi feito e pode tomar decisões coerentes com o estado atual do código.

O conceito de agente

A palavra "agente" tem um significado técnico específico aqui. Um agente é um sistema que percebe o ambiente, toma decisões e age para atingir um objetivo. No caso do Claude Code, o ambiente é o seu projeto: os arquivos, a estrutura de pastas, o histórico de mudanças. O objetivo é o que você descreve em linguagem natural. As ações são as operações que ele executa: ler arquivos, escrever código, rodar comandos.

Na prática, isso significa que quando você diz "crie uma página de login com validação de email e senha", o Claude Code não apenas escreve um bloco de código genérico. Ele primeiro lê os arquivos existentes para entender qual tecnologia o projeto usa, qual é o padrão de organização das pastas, como outros componentes foram escritos. Depois escreve o código novo de forma consistente com o que já existe. Se encontrar um erro durante a execução, ele analisa a mensagem de erro e tenta corrigir. Esse ciclo de percepção, decisão e ação é o que define um agente.

"A diferença entre pedir código a um chatbot e usar o Claude Code é a diferença entre pedir uma planta baixa e contratar um arquiteto que conhece o terreno."

Conceito central do desenvolvimento agêntico

Por que isso importa para quem não sabe programar

Quando você não sabe programar, o maior obstáculo não é a falta de criatividade ou de ideias. É a distância entre o que você quer e o que você consegue expressar em código. Um chatbot reduz essa distância parcialmente: você descreve, ele gera código, você tenta usar. Mas sem saber programar, você não consegue avaliar se o código está correto, não sabe onde colocá-lo, não entende por que ele não funciona quando algo dá errado.

O Claude Code reduz essa distância de forma mais completa porque ele opera no nível do projeto, não do trecho de código. Você não precisa saber onde colocar o arquivo: ele sabe. Você não precisa entender a mensagem de erro: ele lê, interpreta e corrige. Você não precisa conhecer a sintaxe da linguagem: ele escreve código correto e consistente com o restante do projeto. O que você precisa saber é descrever o que quer com clareza suficiente para que o agente entenda a intenção.

Isso não significa que você pode usar o Claude Code sem nenhum envolvimento. Você ainda precisa revisar o que foi feito, testar se o resultado corresponde ao que esperava e tomar decisões sobre a direção do projeto. Mas o trabalho técnico de transformar intenção em código funcional passa a ser responsabilidade do agente, não sua.

Como instalar e configurar o Claude Code

A instalação do Claude Code leva menos de dez minutos. O que leva mais tempo é entender o que cada peça faz, e por isso este capítulo explica cada etapa com contexto.

Pré-requisitos: o que você precisa ter antes

O Claude Code roda em cima do Node.js, uma plataforma que permite executar JavaScript fora do navegador. Você não precisa saber o que isso significa em detalhes: basta instalar. Acesse nodejs.org e baixe a versão LTS (Long Term Support), que é a versão estável recomendada para uso geral. Instale normalmente, como qualquer programa.

Você também precisa de um terminal. No Windows, use o PowerShell ou o Terminal do Windows. No macOS, use o Terminal ou o iTerm2. No Linux, qualquer emulador de terminal funciona. O terminal é a interface onde você vai digitar comandos e ver as respostas do Claude Code. Se nunca usou um terminal antes, não se preocupe: os comandos que você vai usar são poucos e simples.

Instalação passo a passo

Com o Node.js instalado, abra o terminal e digite o seguinte comando:

Terminal 
npm install -g @anthropic-ai/claude-code

O npm é o gerenciador de pacotes do Node.js. O -g significa que o pacote será instalado globalmente, disponível em qualquer pasta do seu computador. Aguarde a instalação terminar. Quando o terminal voltar ao prompt normal, a instalação foi concluída.

Para confirmar que funcionou, digite:

Terminal 
claude --version

Se aparecer um número de versão, está tudo certo.

Primeiro acesso e autenticação

Na primeira vez que você rodar o Claude Code dentro de uma pasta de projeto, ele vai pedir autenticação. Você pode autenticar de duas formas: com uma conta no claude.ai (plano Pro ou acima) ou com uma chave de API da Anthropic. Para a maioria dos usuários iniciantes, a autenticação via conta claude.ai é mais simples. Digite claude no terminal e siga as instruções que aparecem na tela. O processo abre o navegador para você fazer login.

Estrutura básica de um projeto

O Claude Code funciona dentro de uma pasta de projeto. Antes de começar qualquer trabalho, crie uma pasta para o seu projeto e navegue até ela no terminal:

Terminal 
mkdir meu-projeto
cd meu-projeto
claude

A partir desse momento, o Claude Code está ativo dentro da pasta meu-projeto e pode ler e escrever arquivos nela.

O arquivo CLAUDE.md: instruções permanentes para o agente

O CLAUDE.md é um arquivo de texto que você cria na raiz do seu projeto. Ele funciona como um briefing permanente para o Claude Code: tudo que você escrever nele será lido pelo agente no início de cada sessão. É onde você descreve o que o projeto faz, quais tecnologias usa, quais convenções devem ser seguidas e quais comportamentos você espera.

Por exemplo, se o seu projeto é um sistema de gestão em português, você pode escrever no CLAUDE.md que toda a interface deve estar em português, que os preços são armazenados em centavos e que o banco de dados usa PostgreSQL. O agente vai respeitar essas instruções em todas as sessões, sem que você precise repetir a cada vez.

Criar um CLAUDE.md bem escrito é um dos investimentos mais importantes que você pode fazer no início de um projeto. O capítulo 11 mostra como escrever um CLAUDE.md eficaz do zero.

A interface do Claude Code

O terminal parece intimidador para quem nunca usou. Mas a interface do Claude Code é, na prática, uma conversa: você escreve o que quer, ele responde e age.

O terminal como interface principal

Diferente de ferramentas com botões e menus visuais, o Claude Code opera no terminal. Isso não é uma limitação: é uma escolha deliberada. O terminal permite que o agente execute qualquer operação no sistema, leia qualquer arquivo e rode qualquer comando, sem as restrições de uma interface gráfica. Para você, usuário, isso significa que a interação acontece por texto: você digita uma instrução, pressiona Enter e o agente trabalha.

Quando você inicia o Claude Code com o comando claude, aparece um prompt interativo. É ali que você vai digitar suas instruções. O agente responde no mesmo terminal, mostrando o que está fazendo, quais arquivos está lendo ou escrevendo e pedindo confirmação quando necessário.

Comandos básicos do dia a dia

Você não precisa memorizar dezenas de comandos. Os mais usados são poucos:

Comandos essenciais
claude — inicia o Claude Code na pasta atual
claude "sua instrução aqui" — executa uma instrução diretamente, sem abrir o modo interativo
claude --continue — retoma a última conversa de onde parou
claude --resume — lista conversas anteriores para você escolher qual retomar
Ctrl+C — interrompe o que o agente está fazendo no momento

Modos de operação: interativo e autônomo

O Claude Code tem dois modos principais de operação. No modo interativo, você e o agente trabalham em diálogo: ele propõe uma ação, você aprova ou rejeita, ele executa e reporta o resultado. Esse modo é recomendado para quem está começando, porque permite revisar cada passo antes que ele aconteça.

No modo autônomo, você dá uma instrução mais ampla e o agente executa uma sequência de ações sem pedir confirmação a cada passo. Esse modo é útil para tarefas repetitivas ou bem definidas, mas exige mais confiança no agente e mais atenção ao revisar o resultado final. Para ativar o modo autônomo, use a flag --dangerously-skip-permissions, mas leia o capítulo 9 antes de usar essa opção.

Como dar instruções eficazes

A qualidade do resultado do Claude Code depende diretamente da qualidade da instrução que você dá. Instruções vagas produzem resultados genéricos. Instruções específicas produzem resultados precisos. Alguns princípios práticos:

Descreva o comportamento esperado, não apenas o que você quer criar. Em vez de "crie um formulário de contato", diga "crie um formulário de contato com campos de nome, email e mensagem, que valide se o email tem formato correto e exiba uma mensagem de sucesso após o envio". A segunda instrução deixa pouco espaço para interpretação.

Mencione restrições relevantes. Se o projeto usa uma tecnologia específica, diga qual. Se há um padrão visual a seguir, descreva-o. Se há algo que não deve ser alterado, diga explicitamente.

Divida tarefas grandes em partes menores. Em vez de pedir "construa o módulo de vendas completo", comece com "crie a tela de listagem de pedidos" e avance passo a passo. Isso facilita a revisão e reduz o risco de erros acumulados.

Slash commands: atalhos para ações comuns

Dentro do modo interativo, o Claude Code oferece comandos de barra que executam ações específicas rapidamente:

Slash commands úteis
/help — mostra a lista de comandos disponíveis
/clear — limpa o histórico da conversa atual e libera contexto
/compact — comprime o histórico para economizar contexto sem perder o essencial
/status — mostra informações sobre o uso de contexto e tokens
/model — permite trocar o modelo de IA em uso (Opus, Sonnet, Haiku)
/review — solicita uma revisão do código recém-escrito
/init — cria um CLAUDE.md inicial para o projeto atual

O comando /init merece atenção especial: ele analisa o projeto atual e gera automaticamente um CLAUDE.md com as informações básicas detectadas. É um bom ponto de partida para projetos que já têm algum código.

Como o Claude Code lê e entende seu projeto

Antes de escrever uma linha de código, o Claude Code lê. Ele mapeia o projeto, entende a estrutura e só então age. Esse processo de leitura é o que torna o agente capaz de trabalhar de forma coerente.

Como o Claude Code mapeia arquivos e pastas

Quando você inicia o Claude Code em uma pasta, ele não lê todos os arquivos de uma vez. Ele começa pelo CLAUDE.md, se existir, e depois explora a estrutura de pastas para entender a organização do projeto. Conforme você dá instruções, ele lê os arquivos relevantes para aquela tarefa específica.

Por exemplo, se você pede para criar uma nova página em um projeto web, o agente vai ler as páginas existentes para entender o padrão usado, os componentes disponíveis e as convenções de nomenclatura. Ele não vai ler o banco de dados ou os arquivos de configuração de servidor, a menos que sejam relevantes para a tarefa.

Essa leitura seletiva é importante porque o agente tem um limite de quanto pode processar de uma vez, o chamado "contexto". Projetos muito grandes exigem atenção especial a esse limite, assunto tratado no capítulo 10.

O conceito de contexto

Contexto, no vocabulário do Claude Code, é tudo que o agente "vê" em um determinado momento: o histórico da conversa, os arquivos que leu, as instruções do CLAUDE.md e os resultados das ações que executou. Quanto mais contexto, mais informação o agente tem para tomar decisões. Mas o contexto tem um limite físico: quando ele enche, o agente começa a "esquecer" as partes mais antigas da conversa.

Para sessões longas de trabalho, use o comando /compact periodicamente. Ele resume o histórico da conversa, liberando espaço sem perder as informações essenciais. Se a sessão ficar muito longa, considere iniciar uma nova com /clear e retomar o trabalho com uma instrução que resume o estado atual.

Como o CLAUDE.md guia o comportamento do agente

O CLAUDE.md é lido no início de cada sessão e funciona como a memória permanente do projeto. Tudo que você escrever nele vai influenciar como o agente se comporta. Isso inclui: a descrição do projeto, as tecnologias usadas, as convenções de código, as regras de negócio importantes e qualquer instrução específica sobre como o agente deve agir.

Um CLAUDE.md bem escrito reduz drasticamente a quantidade de instruções repetitivas que você precisa dar. Em vez de lembrar ao agente a cada sessão que "os preços são em centavos" ou "a interface deve estar em português", você escreve isso uma vez no CLAUDE.md e ele respeita em todas as sessões seguintes.

Boas práticas para organizar o projeto

A organização do projeto afeta diretamente a qualidade do trabalho do Claude Code. Projetos bem organizados, com pastas claras e nomes de arquivos descritivos, permitem que o agente navegue com mais precisão. Algumas práticas que fazem diferença:

Organização que ajuda o agente
Use nomes de pastas e arquivos em português ou inglês, mas seja consistente: não misture os dois idiomas
Separe claramente o código de interface (frontend) do código de lógica de negócio (backend)
Mantenha um arquivo README.md com uma descrição básica do projeto
Evite arquivos muito grandes: prefira dividir em arquivos menores e bem nomeados
Use Git desde o início, mesmo que não entenda todos os detalhes (o capítulo 11 explica como)

Permissões de leitura e escrita

Por padrão, o Claude Code pede confirmação antes de escrever ou modificar arquivos. Quando ele propõe uma mudança, você vê exatamente o que vai ser alterado antes de aprovar. Isso é chamado de "diff": uma visualização que mostra em verde o que será adicionado e em vermelho o que será removido.

Você pode configurar permissões mais amplas para pastas específicas, permitindo que o agente escreva sem pedir confirmação a cada arquivo. Isso acelera o trabalho em tarefas repetitivas, mas exige que você revise o resultado com atenção ao final. As configurações de permissão ficam no arquivo .claude/settings.json dentro do projeto.

Criando software do zero com Claude Code

Criar software com o Claude Code não é apertar um botão e esperar o resultado pronto. É um processo iterativo: você descreve, revisa, corrige e avança. Entender esse ciclo é o que separa quem obtém bons resultados de quem fica frustrado.

O fluxo de trabalho básico

O processo de criar software com o Claude Code segue três etapas que se repetem: descrever, revisar e aprovar. Você descreve o que quer em linguagem natural. O agente propõe uma solução, mostrando quais arquivos vai criar ou modificar. Você revisa a proposta e aprova, rejeita ou pede ajustes. Depois de aprovado, o agente executa e reporta o resultado.

Esse ciclo parece simples, mas a qualidade de cada etapa determina o resultado final. Uma descrição vaga leva a uma proposta genérica. Uma revisão superficial deixa passar erros. Uma aprovação sem entender o que foi feito acumula problemas que aparecem mais tarde.

Como descrever o que você quer

A instrução que você dá ao Claude Code é o ponto de partida de tudo. Quanto mais precisa, melhor o resultado. Algumas perguntas que ajudam a formular uma boa instrução:

O que deve existir ao final? Descreva o resultado esperado, não o processo. "Quero uma tela que liste todos os clientes cadastrados, com nome, email e data de cadastro, e um botão para adicionar novo cliente" é melhor do que "crie uma tela de clientes".

Quais são as regras de negócio? Se há validações, restrições ou comportamentos específicos, mencione-os. "O email deve ser único no sistema" ou "só usuários com perfil de administrador podem excluir registros" são informações que o agente não vai inventar: ele precisa que você forneça.

O que não deve mudar? Se há partes do projeto que não devem ser tocadas, diga explicitamente. "Não altere o arquivo de configuração do banco de dados" ou "mantenha o estilo visual das outras telas" evita surpresas.

O ciclo de iteração

Raramente o primeiro resultado é o final. O fluxo normal é: você pede, o agente entrega, você testa, encontra algo que não está certo, pede ajuste, o agente corrige. Esse ciclo pode se repetir várias vezes para uma mesma funcionalidade, e isso é normal. Não é sinal de que o agente falhou: é o processo natural de refinamento.

A chave é ser específico nos ajustes. Em vez de "isso não ficou bom", diga "o botão de salvar está aparecendo antes do formulário ser preenchido, ele deve ficar desabilitado até que todos os campos obrigatórios tenham conteúdo". Quanto mais preciso o feedback, mais rápido o ajuste.

Como lidar com erros

Erros acontecem. O Claude Code vai encontrar situações onde o código que escreveu não funciona como esperado. Quando isso acontece, o agente normalmente detecta o erro, analisa a mensagem e tenta corrigir automaticamente. Se não conseguir resolver sozinho, ele vai descrever o problema e pedir orientação.

Para você, a postura correta diante de um erro é não entrar em pânico. Leia o que o agente reportou, mesmo que não entenda todos os detalhes técnicos. Muitas vezes a solução é simples: uma dependência que precisa ser instalada, uma configuração que está faltando, um arquivo que precisa ser criado. O agente vai guiar você.

A importância de revisar mesmo sem saber programar

Você não precisa entender o código para revisar o resultado. O que você precisa revisar é o comportamento: a tela aparece como esperado? O botão faz o que deveria? Os dados são salvos corretamente? Essa revisão funcional é sua responsabilidade, e ela é insubstituível.

O Claude Code pode escrever código tecnicamente correto que não faz o que você queria, simplesmente porque a instrução foi ambígua. Testar o resultado é a única forma de confirmar que a intenção foi traduzida corretamente.

Atenção: vibe coding e seus riscos

O termo "vibe coding" descreve a prática de usar IA para criar software sem revisar o que foi produzido, confiando cegamente no resultado. É tentador, especialmente quando o agente parece confiante. Mas código não revisado acumula problemas: falhas de segurança, lógica incorreta, dados corrompidos. Use o Claude Code como parceiro, não como substituto do seu julgamento. Revise sempre o que foi feito antes de colocar em produção.

Skills: expandindo as capacidades do agente

O Claude Code, por padrão, já faz muito. Mas seu comportamento pode ser expandido com skills: conjuntos de instruções especializadas que ensinam o agente a executar tarefas específicas com mais precisão e profundidade.

O que são skills

Skills são arquivos de instrução que você instala no Claude Code para adicionar capacidades especializadas. Pense nelas como manuais de procedimento: quando você instala uma skill de revisão de segurança, por exemplo, o agente passa a saber exatamente quais aspectos verificar, quais padrões de vulnerabilidade procurar e como reportar os problemas encontrados. Sem a skill, ele faria uma revisão genérica. Com ela, faz uma revisão estruturada e completa.

Skills não são plugins no sentido técnico: elas não adicionam código novo ao Claude Code. São arquivos de texto com instruções detalhadas que o agente lê e segue. Isso significa que você pode criar suas próprias skills para tarefas específicas do seu negócio, como "revisar contratos segundo as regras da empresa" ou "gerar relatórios no formato padrão do cliente X".

Como as skills expandem as capacidades padrão

O Claude Code sem skills já consegue escrever código, corrigir erros e organizar projetos. Com skills, ele passa a executar fluxos de trabalho completos com consistência. Uma skill de desenvolvimento frontend, por exemplo, pode incluir instruções sobre acessibilidade, performance, padrões de componentes e testes visuais. O agente vai seguir todas essas diretrizes automaticamente, sem que você precise lembrar de cada uma.

Como instalar uma skill

A instalação de uma skill é feita pelo próprio Claude Code. Dentro de uma sessão interativa, use o comando:

Terminal 
/install-skill nome-da-skill

O agente vai buscar a skill no repositório oficial, baixar o arquivo de instrução e instalá-lo na pasta .claude/skills/ do seu projeto. A partir desse momento, a skill está disponível para uso.

Você também pode instalar skills manualmente: basta criar um arquivo .md dentro da pasta .claude/skills/ com as instruções que você quer que o agente siga. Esse arquivo pode ser escrito em linguagem natural, sem nenhum conhecimento técnico.

Como usar uma skill depois de instalada

Para invocar uma skill, use o comando de barra com o nome dela dentro de uma sessão interativa:

Terminal 
/nome-da-skill

O agente vai executar o fluxo definido pela skill no contexto atual do projeto. Algumas skills são invocadas diretamente; outras funcionam como modificadores de comportamento que ficam ativos durante toda a sessão.

Skills do ecossistema oficial

MCP: conectando o Claude Code ao mundo real

O Claude Code, por padrão, trabalha com os arquivos do seu projeto. O MCP quebra essa fronteira: ele conecta o agente a bancos de dados, navegadores, APIs externas e qualquer outro sistema que você precisar.

O que é o MCP e por que ele existe

MCP é a sigla para Model Context Protocol, um protocolo criado pela Anthropic para padronizar a forma como modelos de IA se conectam a ferramentas e fontes de dados externas. Antes do MCP, cada integração era feita de forma personalizada: quem quisesse conectar o Claude a um banco de dados precisava escrever código específico para isso. Com o MCP, existe um padrão único que qualquer ferramenta pode implementar, e o Claude Code sabe como se comunicar com qualquer ferramenta que siga esse padrão.

O resultado prático é que o ecossistema de integrações cresceu rapidamente. Hoje existem servidores MCP para bancos de dados relacionais, sistemas de arquivos, navegadores web, plataformas de comunicação, APIs de serviços populares e muito mais. Você instala o servidor MCP correspondente à ferramenta que quer usar, e o Claude Code passa a ter acesso a ela.

A analogia certa: MCP como tomada universal

Imagine que o Claude Code é um aparelho elétrico. Por padrão, ele funciona com a energia que já tem: os arquivos do projeto. O MCP é como uma tomada universal que permite conectar esse aparelho a qualquer fonte de energia externa: um banco de dados, um serviço web, um sistema de arquivos remoto. Cada servidor MCP é um adaptador diferente, mas todos usam o mesmo padrão de encaixe.

Essa analogia ajuda a entender por que o MCP é importante: ele transforma o Claude Code de uma ferramenta que trabalha com arquivos locais em um agente capaz de interagir com sistemas reais em produção.

Como o MCP funciona por baixo dos panos

Um servidor MCP é um programa que roda em segundo plano no seu computador. Ele expõe um conjunto de "ferramentas" que o Claude Code pode usar: consultar um banco de dados, abrir uma página no navegador, enviar uma mensagem, ler um arquivo remoto. Quando você dá uma instrução ao Claude Code que requer uma dessas ferramentas, ele se comunica com o servidor MCP correspondente, que executa a ação e retorna o resultado.

Você não precisa entender os detalhes técnicos dessa comunicação. O que importa é que, do seu ponto de vista, o agente simplesmente passa a ter acesso a mais coisas. Você pode pedir "consulte o banco de dados e me mostre os pedidos do último mês" e o agente vai fazer isso, desde que o servidor MCP do banco de dados esteja configurado.

Como instalar um servidor MCP

A instalação de um servidor MCP é feita em duas etapas. Primeiro, você instala o pacote do servidor. Segundo, você registra o servidor na configuração do Claude Code.

Exemplo com o servidor MCP para PostgreSQL:

Passo 1: instalar o servidor 
npm install -g @modelcontextprotocol/server-postgres
Passo 2: registrar no Claude Code 
claude mcp add postgres -- npx @modelcontextprotocol/server-postgres postgresql://usuario:senha@localhost/banco

Após o registro, o Claude Code vai reconhecer automaticamente as ferramentas disponibilizadas pelo servidor e poderá usá-las nas próximas sessões.

Navegador (Puppeteer, Playwright) — permite abrir páginas web, preencher formulários e capturar screenshots
Sistema de arquivos — expande o acesso a pastas fora do projeto atual

Diferença entre skills e MCP

Skills e MCP são complementares, mas servem a propósitos diferentes. Skills ensinam o agente como fazer algo: como revisar segurança, como estruturar testes, como organizar componentes. MCP dá ao agente acesso a algo: um banco de dados, um serviço externo, um sistema de arquivos remoto.

Use skills quando quiser que o agente siga um processo específico ou adote um padrão de trabalho. Use MCP quando quiser que o agente interaja com sistemas externos ao projeto. Em muitos casos, você vai usar os dois juntos: uma skill que define como fazer uma tarefa, e um servidor MCP que dá acesso aos dados necessários para executá-la.

Integrações e o ecossistema do Claude Code

O Claude Code não existe em isolamento. Ele se integra a editores de código, plataformas de versionamento e serviços externos, formando um ecossistema que cresce a cada mês.

Integração com editores de código

O Claude Code pode ser usado diretamente no terminal, mas também se integra aos editores de código mais populares. No VS Code (Visual Studio Code), existe uma extensão oficial que embute o Claude Code dentro do editor: você pode dar instruções ao agente sem sair do ambiente onde o código está aberto. O mesmo vale para os editores da família JetBrains, como IntelliJ IDEA, PyCharm e WebStorm.

Para quem não sabe programar, essa integração tem um benefício prático: você pode ver o código sendo escrito em tempo real, no mesmo lugar onde ele vai existir. Não é necessário alternar entre o terminal e o editor para acompanhar o que está sendo feito.

A instalação da extensão é feita pela loja de extensões do próprio editor. Busque por "Claude Code" e instale a extensão oficial da Anthropic. Após a instalação, a autenticação é compartilhada com a instalação de linha de comando: você não precisa fazer login duas vezes.

Integração com GitHub

O GitHub é a plataforma mais usada para armazenar e versionar código. O Claude Code se integra a ele de duas formas. A primeira é via MCP: com o servidor MCP do GitHub configurado, o agente pode criar branches, abrir pull requests, comentar em issues e executar outras ações diretamente da linha de comando. A segunda é via a skill github-pr-review, que automatiza o processo de revisão de código: o agente lê os comentários de revisores, classifica por severidade e aplica as correções necessárias.

Para quem não sabe programar, a integração com GitHub é especialmente útil para manter um histórico seguro do projeto. O capítulo 11 explica como usar Git e GitHub com a ajuda do Claude Code, sem precisar entender os detalhes técnicos.

O conceito de hooks

Hooks são ações automáticas que o Claude Code executa em resposta a eventos específicos. Por exemplo: sempre que o agente terminar de escrever código, execute automaticamente os testes. Ou: antes de qualquer commit, verifique se há problemas de segurança. Ou: após cada sessão, gere um resumo do que foi feito.

Os hooks são configurados no arquivo .claude/settings.json e podem executar qualquer comando do sistema. Para quem não sabe programar, a forma mais simples de configurar hooks é pedir ao próprio Claude Code que os configure: descreva o comportamento que você quer e ele vai escrever a configuração necessária.

Exemplo de instrução para configurar um hook 
# Dentro de uma sessão do Claude Code:
Configure um hook que execute os testes automaticamente
sempre que você terminar de escrever ou modificar código.

Segurança, permissões e boas práticas

O Claude Code tem acesso ao seu sistema de arquivos e pode executar comandos no terminal. Isso é poderoso, e por isso exige atenção. Este capítulo explica como usar essa capacidade com segurança.

O que o Claude Code pode e não pode fazer por padrão

Por padrão, o Claude Code pode ler qualquer arquivo dentro da pasta do projeto, escrever e modificar arquivos (com sua confirmação), executar comandos no terminal (com sua confirmação) e acessar a internet para buscar documentação ou instalar dependências. O que ele não faz por padrão: não acessa arquivos fora da pasta do projeto sem permissão explícita, não executa ações irreversíveis sem confirmação e não envia dados do seu projeto para servidores externos além da API da Anthropic, que processa as instruções.

Como funcionam as permissões

O sistema de permissões do Claude Code funciona em camadas. A camada mais básica é a confirmação manual: antes de escrever um arquivo ou executar um comando, o agente mostra o que vai fazer e espera você aprovar. Você pode expandir as permissões para pastas específicas, permitindo que o agente escreva sem pedir confirmação a cada arquivo. E pode restringir ainda mais, bloqueando o acesso a pastas sensíveis como as que contêm credenciais ou configurações de produção.

As permissões são configuradas no arquivo .claude/settings.json. Para a maioria dos projetos, as configurações padrão são adequadas. Ajuste apenas quando tiver uma razão específica.

Quando deixar o agente autônomo e quando supervisionar

A decisão de quanto autonomia dar ao agente depende do tipo de tarefa e do estágio do projeto. Para tarefas de criação em projetos novos, onde não há risco de sobrescrever trabalho existente, mais autonomia acelera o desenvolvimento. Para tarefas que modificam código em produção, que alteram configurações críticas ou que envolvem dados reais de usuários, supervisione cada passo.

Uma regra prática: quanto mais irreversível a ação, mais supervisão ela merece. Criar um novo arquivo é reversível: você pode deletar. Modificar o banco de dados de produção pode não ser. Calibre a autonomia de acordo com o risco.

O que nunca deixar o Claude Code fazer sem revisão

Ações que sempre exigem revisão

Modificações em banco de dados de produção: qualquer alteração em dados reais de usuários deve ser revisada e testada em ambiente de desenvolvimento primeiro.

Alterações em arquivos de configuração de segurança: senhas, chaves de API, certificados e configurações de autenticação nunca devem ser modificados sem revisão cuidadosa.

Deploy para produção: o processo de colocar software em funcionamento para usuários reais deve sempre ter uma etapa de revisão humana.

Exclusão de arquivos ou dados: operações de exclusão são irreversíveis. Confirme sempre antes de aprovar.

Boas práticas para projetos reais

Use Git desde o início do projeto. O Git é um sistema de controle de versão que registra cada mudança feita no código, permitindo voltar a qualquer estado anterior. Se o Claude Code fizer algo errado, você pode desfazer com um comando. O capítulo 11 explica como usar Git com a ajuda do próprio agente.

Nunca armazene senhas ou chaves de API diretamente no código. Use variáveis de ambiente: arquivos separados que contêm informações sensíveis e não são incluídos no controle de versão. O Claude Code sabe como configurar isso e vai sugerir a abordagem correta se você mencionar que tem credenciais para armazenar.

Faça backups regulares. Mesmo com Git, um backup externo do projeto completo é uma camada adicional de segurança. Serviços como GitHub, GitLab ou Bitbucket funcionam como backup remoto automático quando você faz push regularmente.

Erros comuns de quem está começando

O erro mais comum é aprovar ações sem ler o que o agente propõe. A pressa de ver o resultado leva a aprovações automáticas que acumulam problemas. Leia sempre o que o agente vai fazer antes de confirmar.

O segundo erro mais comum é trabalhar diretamente em produção. Sempre desenvolva em um ambiente separado (chamado de ambiente de desenvolvimento ou staging) e só transfira para produção após testar. O Claude Code pode ajudar a configurar esses ambientes.

Limites, custos e o que esperar

O Claude Code é uma ferramenta capaz, mas tem limites reais. Conhecê-los antes de começar evita frustrações e ajuda a planejar melhor o uso.

O que o Claude Code faz bem

O Claude Code se destaca em tarefas bem definidas com contexto claro. Criar componentes de interface, escrever lógica de negócio, configurar integrações com APIs, estruturar bancos de dados, escrever testes automatizados, refatorar código existente: tudo isso o agente faz com alta qualidade quando as instruções são precisas e o projeto está bem organizado.

Ele também é eficaz em tarefas de análise: revisar código em busca de problemas, explicar o que um trecho de código faz, sugerir melhorias de performance ou segurança. Para quem não sabe programar, essa capacidade de análise é especialmente valiosa: você pode pedir ao agente que explique em linguagem simples o que qualquer parte do projeto faz.

Onde o Claude Code ainda falha

O agente tem dificuldade com requisitos ambíguos ou contraditórios. Se você não sabe exatamente o que quer, o resultado vai refletir essa incerteza. Projetos sem estrutura clara, sem CLAUDE.md e sem convenções definidas produzem resultados inconsistentes.

Tarefas que exigem conhecimento de negócio muito específico também são um ponto fraco. O agente não conhece as regras tributárias do seu setor, os requisitos legais da sua indústria ou as particularidades do seu mercado. Você precisa fornecer essas informações explicitamente.

Por fim, o agente pode cometer erros em código complexo que envolve múltiplos sistemas interagindo. Quanto mais partes móveis, maior a chance de algo não funcionar como esperado na primeira tentativa.

Limites de contexto em projetos grandes

O contexto é a quantidade de informação que o agente pode processar de uma vez. Projetos muito grandes, com centenas de arquivos e milhares de linhas de código, podem exceder esse limite. Quando isso acontece, o agente começa a trabalhar com informação incompleta, o que pode levar a decisões inconsistentes.

Para projetos grandes, a estratégia é dividir o trabalho em sessões focadas: cada sessão trata de uma parte específica do projeto, com um CLAUDE.md que descreve o contexto relevante para aquela parte. O comando /compact também ajuda a gerenciar o contexto dentro de uma sessão longa.

Modelo de cobrança

O Claude Code pode ser usado de duas formas: com uma assinatura do claude.ai (plano Pro ou Max) ou com uma chave de API da Anthropic com cobrança por uso. No plano Pro, o uso do Claude Code está incluído dentro dos limites mensais da assinatura. No modelo de API, você paga por token processado: cada instrução que você dá e cada resposta que o agente gera consome tokens, e o custo depende do volume.

Para projetos pequenos e médios, o plano Pro costuma ser suficiente. Para uso intensivo, com muitas sessões longas por dia, o modelo de API pode ser mais econômico dependendo do volume. A Anthropic disponibiliza uma calculadora de custos na documentação oficial para ajudar a estimar.

Quando o Claude Code não é a ferramenta certa

Para projetos muito simples, como uma página estática ou um formulário básico, ferramentas como Webflow, Notion ou Google Sites podem ser mais rápidas e não exigem nenhuma configuração técnica. O Claude Code brilha em projetos com lógica de negócio real, integrações entre sistemas e necessidade de personalização que ferramentas no-code não conseguem atender.

Se o projeto exige conformidade regulatória rigorosa, como sistemas médicos ou financeiros com certificações específicas, o Claude Code pode ajudar no desenvolvimento, mas a validação e certificação ainda exigem profissionais especializados.

Fluxo de trabalho completo para não programadores

Este capítulo reúne tudo que foi apresentado nos anteriores em um fluxo de trabalho prático: do zero ao software funcionando, com cada etapa descrita de forma que qualquer pessoa possa seguir.

Antes de abrir o terminal: estruture suas ideias

O trabalho mais importante acontece antes de digitar qualquer comando. Você precisa ter clareza sobre o que está construindo. Isso não significa escrever uma especificação técnica completa: significa responder a algumas perguntas básicas.

Qual é o problema que o software resolve? Quem vai usar? Quais são as ações principais que o usuário vai executar? Quais dados precisam ser armazenados? Existem integrações com outros sistemas? Há regras de negócio específicas, como cálculos, validações ou fluxos de aprovação?

Escreva as respostas em um documento simples, sem se preocupar com formato. Esse documento vai ser a base do seu CLAUDE.md e das suas primeiras instruções ao agente.

Como escrever um CLAUDE.md eficaz

Um bom CLAUDE.md tem quatro seções principais. A primeira é a descrição do projeto: o que é, para quem é e qual problema resolve. A segunda é a stack tecnológica: quais tecnologias o projeto usa (ou deve usar). Se você não sabe qual tecnologia escolher, peça ao Claude Code uma recomendação antes de criar o CLAUDE.md. A terceira é as convenções: idioma da interface, formato de datas, moeda, padrões de nomenclatura. A quarta é as regras de negócio críticas: as informações que o agente precisa conhecer para tomar decisões corretas.

Exemplo de CLAUDE.md para um sistema de gestão 
# Sistema de Gestão de Pedidos

## Sobre o projeto
Sistema interno para gerenciar pedidos de uma loja de móveis.
Permite cadastrar clientes, registrar pedidos, acompanhar
status de entrega e gerar relatórios mensais.

## Tecnologias
- Next.js 15 com App Router
- PostgreSQL via Supabase
- Tailwind CSS para estilo
- TypeScript

## Convenções
- Interface em português brasileiro
- Preços em centavos (inteiro)
- Datas no formato DD/MM/AAAA
- Toda interface deve ser responsiva

## Regras de negócio
- Pedido só pode ser cancelado se status for "aguardando"
- Desconto máximo permitido: 15%
- Frete grátis para pedidos acima de R$ 500

Como usar Git sem entender de programação

Git é um sistema que registra cada mudança feita no projeto, como um histórico detalhado que permite voltar a qualquer ponto anterior. Você não precisa entender como ele funciona internamente: o Claude Code pode executar os comandos Git por você.

Os três momentos em que você vai usar Git são: ao iniciar o projeto (para criar o repositório), ao final de cada sessão de trabalho (para salvar o progresso) e antes de qualquer mudança grande (para ter um ponto de retorno seguro). Para cada um desses momentos, basta pedir ao agente:

Instruções Git para o Claude Code 
# Ao iniciar o projeto:
Inicialize um repositório Git neste projeto e faça o
primeiro commit com todos os arquivos atuais.

# Ao final de uma sessão:
Faça um commit com as mudanças desta sessão. Use uma
mensagem descritiva sobre o que foi feito.

# Antes de uma mudança grande:
Crie um commit de checkpoint antes de começarmos
a refatorar o módulo de pagamentos.

Como testar o que foi criado

Testar software não exige conhecimento técnico: exige atenção e método. Para cada funcionalidade criada, percorra o caminho que um usuário real percorreria. Preencha formulários com dados válidos e inválidos. Tente ações que não deveriam ser permitidas. Verifique se as mensagens de erro fazem sentido. Confirme que os dados são salvos e recuperados corretamente.

Além dos testes manuais, peça ao Claude Code que escreva testes automatizados para as funcionalidades críticas. Testes automatizados são programas que verificam se o software funciona corretamente de forma repetível. Você não precisa escrever esses testes: o agente faz isso. Mas você precisa rodá-los regularmente e investigar quando algum falhar.

Como manter e evoluir o projeto

Software não é um produto acabado: é um sistema vivo que precisa de manutenção e evolução. Para manter o projeto saudável ao longo do tempo, adote algumas práticas regulares.

Documente as decisões importantes no CLAUDE.md. Quando você decidir usar uma tecnologia específica, adotar uma convenção ou implementar uma regra de negócio, registre no CLAUDE.md o que foi decidido e por quê. Isso garante que o agente (e você mesmo, meses depois) entenda o contexto das escolhas.

Revise o CLAUDE.md periodicamente. À medida que o projeto evolui, as informações no CLAUDE.md podem ficar desatualizadas. Reserve um tempo a cada mês para revisar e atualizar o arquivo.

Mantenha o repositório Git atualizado. Faça commits frequentes, com mensagens descritivas. Isso cria um histórico que permite entender o que mudou e quando, e facilita a recuperação em caso de problemas.
