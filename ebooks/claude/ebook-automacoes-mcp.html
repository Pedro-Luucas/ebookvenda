Introdução

Automações que trabalham por você

Toda semana, você repete as mesmas tarefas. Abre o e-mail, verifica a agenda, anota um gasto, registra uma ideia em algum lugar que provavelmente vai esquecer. São tarefas pequenas, mas somadas consomem tempo e atenção que poderiam estar em outro lugar.

Automações pessoais existem para resolver exatamente isso: fazer com que tarefas repetitivas aconteçam sozinhas, sem que você precise iniciar cada uma delas manualmente. Não é um conceito novo. O que mudou é que, até pouco tempo atrás, criar automações exigia saber programar ou pagar por ferramentas como Zapier ou Make.

O Claude Code com MCP muda essa equação. A diferença em relação a ferramentas como Zapier não é só de custo: é de inteligência. Zapier conecta campos. O Claude Code entende contexto. Você não precisa mapear "campo A vai para campo B". Você descreve o que quer em linguagem natural, e o agente descobre como fazer.

Este ebook ensina como usar o Claude Code, junto com servidores MCP (Model Context Protocol, o protocolo que permite ao Claude Code se conectar a ferramentas externas), para criar automações pessoais que integram as ferramentas do seu dia a dia: Telegram, Google Calendar, Gmail, Google Sheets e Notion. Você não precisa saber programar. Toda automação é criada pelo próprio Claude Code via MCP, sem scripts manuais sempre que possível.

O que você vai ser capaz de fazer

Ao final deste ebook, você vai saber como receber um resumo da sua agenda toda manhã direto no Telegram, como registrar gastos enviando uma mensagem de texto, como capturar ideias no Notion sem abrir o aplicativo, como fazer o Claude Code ler seus e-mails e classificar o que é urgente, e como combinar tudo isso em rotinas automáticas que rodam sem intervenção.

Você também vai aprender a rodar essas automações na nuvem, de forma gratuita, para que funcionem mesmo quando seu computador estiver desligado.

Uma nota sobre o WhatsApp

O WhatsApp é o aplicativo de mensagens mais usado no Brasil, mas a integração com bots exige a API oficial do WhatsApp Business, que tem custo por mensagem enviada e processo de aprovação. Por isso, este ebook usa o Telegram como canal de mensagens principal. O Telegram tem API aberta, gratuita, sem aprovação necessária, e é ideal para bots pessoais. Se você ainda não usa o Telegram, criar uma conta leva menos de dois minutos.

Pré-requisitos

Este ebook assume que você já tem o Claude Code instalado e sabe como instalar servidores MCP. Se ainda não chegou nesse ponto, o ebook Claude Code: O Guia Completo cobre esses passos em detalhes. Aqui, o ponto de partida é: Claude Code funcionando, terminal aberto, pronto para criar.

Como pensar em automações antes de criar

Uma automação mal planejada não economiza tempo. Ela só transfere o problema para outro lugar.

O que é uma automação, em linguagem simples

Uma automação é uma tarefa que acontece sozinha. Em vez de você abrir o Google Calendar toda manhã para ver o que tem no dia, a automação faz isso por você e manda o resultado no Telegram. Em vez de você anotar um gasto manualmente na planilha, você envia uma mensagem e a automação registra.

O mecanismo por trás é sempre o mesmo: algo dispara a automação (um horário, uma mensagem, um evento), e a automação executa uma ou mais ações em resposta. Esse par, gatilho e ação, é a estrutura de toda automação que você vai criar neste ebook.

Os três tipos de automação cobertos neste ebook

Tipos de acionamento
Via terminal: você abre o terminal e digita uma instrução ao Claude Code. É o modo mais simples, útil para tarefas que você quer executar agora, sob demanda.
Via Telegram: você envia uma mensagem ao seu bot pessoal, e o bot aciona o Claude Code. Funciona de qualquer lugar, pelo celular, sem precisar abrir o computador.
Via agendamento: a automação roda sozinha em horário fixo, sem nenhuma intervenção. Ideal para resumos diários, relatórios semanais e tarefas recorrentes.

Como identificar o que vale a pena automatizar

Nem toda tarefa merece uma automação. Criar uma automação tem um custo inicial de configuração. Se a tarefa acontece raramente ou muda muito de uma vez para outra, o esforço de automatizar pode não compensar.

As melhores candidatas a automação têm três características: são repetitivas (acontecem toda semana ou todo dia), são previsíveis (seguem sempre o mesmo padrão) e são chatas (você não ganha nada fazendo manualmente, só perde tempo).

Gatilho e ação: a estrutura de toda automação

Antes de pedir ao Claude Code para criar qualquer automação, você precisa ter claro na cabeça qual é o gatilho e qual é a ação. O gatilho é o que dispara a automação. A ação é o que ela faz.

Exemplos concretos: o gatilho é "todo dia às 7h da manhã" e a ação é "ler os eventos do Google Calendar e mandar um resumo no Telegram". Ou: o gatilho é "eu enviar a mensagem 'gastei R$50 no mercado' no Telegram" e a ação é "registrar esse gasto na planilha do Google Sheets".

Como descrever uma automação ao Claude Code

O Claude Code entende linguagem natural, mas quanto mais precisa for a sua descrição, melhor o resultado. Uma boa instrução tem quatro elementos: o que deve acontecer, quando deve acontecer, quais ferramentas estão envolvidas e qual deve ser o formato do resultado.

Instrução vaga (evite)

"Me manda um resumo da agenda no Telegram."

Instrução precisa (use)

"Todo dia às 7h da manhã, leia os eventos do meu Google Calendar para o dia atual e mande uma mensagem formatada no meu bot do Telegram com: saudação com o dia da semana, lista dos eventos com horário e título, e uma linha final com o total de compromissos."

A segunda instrução não exige que você saiba programar. Ela só exige que você saiba o que quer. O Claude Code cuida do resto.

Os erros mais comuns de quem automatiza sem planejar

O erro mais frequente é criar uma automação sem definir o que fazer quando algo der errado. E algo sempre pode dar errado: o MCP perde a autenticação, o Google Calendar não tem eventos naquele dia, o Telegram não recebe a mensagem. Uma automação bem planejada prevê esses casos e define um comportamento para eles.

O segundo erro é automatizar demais de uma vez. Começar com uma automação simples, testar, ajustar e só depois adicionar complexidade é sempre mais eficiente do que tentar criar uma rotina completa de uma vez e depurar dez coisas ao mesmo tempo.

O terceiro erro é não documentar. Quando você cria uma automação hoje e ela para de funcionar em três meses, você vai querer saber o que ela fazia e por quê. Um arquivo CLAUDE.md com uma linha descrevendo cada automação resolve esse problema.

Regra prática

Comece com uma automação. Faça ela funcionar bem. Só então crie a próxima. Automações simples que funcionam valem mais do que rotinas complexas que quebram.

Configurando o ambiente de automações

Antes de criar qualquer automação, o ambiente precisa estar organizado. Um MCP mal configurado vai travar tudo na hora errada.

Como organizar os MCPs no arquivo .mcp.json

O arquivo .mcp.json fica na raiz do seu projeto e lista todos os servidores MCP que o Claude Code pode usar. Para um projeto de automações pessoais, você vai ter vários MCPs configurados ao mesmo tempo: Telegram, Google Calendar, Gmail, Google Sheets e Notion.

A estrutura do arquivo é sempre a mesma: cada MCP tem um nome, um comando para iniciar o servidor e as variáveis de ambiente que ele precisa para funcionar. Veja um exemplo de como o arquivo fica com todos os MCPs deste ebook configurados:

Exemplo de .mcp.json para automações pessoais 
{
  "mcpServers": {
    "telegram": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-telegram"],
      "env": {
        "TELEGRAM_BOT_TOKEN": "seu_token_aqui"
      }
    },
    "google-calendar": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-google-calendar"],
      "env": {
        "GOOGLE_CLIENT_ID": "seu_client_id",
        "GOOGLE_CLIENT_SECRET": "seu_client_secret",
        "GOOGLE_REFRESH_TOKEN": "seu_refresh_token"
      }
    },
    "gmail": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-gmail"],
      "env": {
        "GOOGLE_CLIENT_ID": "seu_client_id",
        "GOOGLE_CLIENT_SECRET": "seu_client_secret",
        "GOOGLE_REFRESH_TOKEN": "seu_refresh_token"
      }
    },
    "google-sheets": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-google-sheets"],
      "env": {
        "GOOGLE_CLIENT_ID": "seu_client_id",
        "GOOGLE_CLIENT_SECRET": "seu_client_secret",
        "GOOGLE_REFRESH_TOKEN": "seu_refresh_token"
      }
    },
    "notion": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-notion"],
      "env": {
        "NOTION_API_KEY": "sua_chave_aqui"
      }
    }
  }
}

O conceito de credenciais: tokens e chaves de API

Cada ferramenta que você conecta ao Claude Code precisa de uma forma de confirmar que você tem permissão para acessá-la. Essa confirmação é feita por meio de credenciais, que podem ser um token (uma sequência de letras e números que funciona como uma senha) ou uma chave de API (o mesmo conceito, com outro nome).

Você não precisa entender como esses tokens funcionam por dentro. O que você precisa saber é: onde encontrar cada um, como colocá-los no arquivo .mcp.json e que eles devem ser tratados como senhas. Nunca compartilhe um token em público, nunca coloque em um repositório aberto no GitHub.

Onde encontrar as credenciais de cada ferramenta
Telegram: o token do bot é gerado pelo BotFather, dentro do próprio Telegram. O Capítulo 3 mostra o passo a passo.
Google Calendar, Gmail e Sheets: as credenciais vêm do Google Cloud Console. Você cria um projeto, ativa as APIs necessárias e gera um Client ID e Client Secret. O processo é o mesmo para as três ferramentas Google.
Notion: a chave de API é gerada em notion.so/my-integrations. Você cria uma integração, copia a chave e conecta ao workspace que quer usar.

Como verificar se cada MCP está funcionando

Antes de criar qualquer automação, confirme que cada MCP responde corretamente. A forma mais simples é pedir ao Claude Code uma tarefa básica com cada ferramenta: listar os eventos de hoje no Calendar, verificar quantos e-mails não lidos existem no Gmail, ler o título da primeira linha de uma planilha no Sheets.

Se o MCP responder com dados reais, está funcionando. Se retornar um erro de autenticação, o problema está nas credenciais. Se não responder nada, o servidor MCP pode não ter iniciado corretamente.

Como estruturar o CLAUDE.md para automações

O arquivo CLAUDE.md é lido pelo Claude Code toda vez que você inicia uma sessão. Para um projeto de automações, ele deve conter: quais MCPs estão configurados, qual é o ID do seu chat no Telegram, quais são os IDs das planilhas e bases de dados que você usa, e uma lista das automações existentes com uma linha descrevendo cada uma.

Exemplo completo de CLAUDE.md para automações pessoais 
# Projeto: Automações Pessoais

## MCPs configurados
- telegram: bot pessoal para notificações e acionamento
- google-calendar: agenda principal (conta pessoal)
- gmail: caixa de entrada principal
- google-sheets: planilha de gastos e registros
- notion: base de conhecimento pessoal

## IDs importantes
- TELEGRAM_CHAT_ID: 123456789
- SHEETS_GASTOS_ID: 1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms
- NOTION_DATABASE_TAREFAS: abc123def456...

## Automações ativas
- resumo_diario: todo dia às 7h, Calendar + Telegram
- registro_gastos: acionada pelo Telegram, grava no Sheets
- captura_ideias: acionada pelo Telegram, cria página no Notion
- resumo_emails: toda segunda às 8h, Gmail + Telegram

## Regras
- Sempre confirmar antes de enviar mensagem no Telegram
- Nunca deletar eventos do Calendar sem confirmação explícita
- Registrar gastos sempre com data, categoria e valor

Telegram: seu painel de controle pessoal

O Telegram é o ponto de entrada e saída de quase todas as automações deste ebook. É por onde você aciona e por onde recebe os resultados.

Por que o Telegram é o centro das automações

O Telegram permite criar bots com uma API completamente gratuita, sem aprovação, sem custo por mensagem e sem limite de uso para bots pessoais. Um bot do Telegram é simplesmente um número de telefone virtual que recebe e envia mensagens. Você cria o bot, conecta ao Claude Code via MCP, e passa a ter um canal direto entre o seu celular e o agente.

Na prática, isso significa que você pode acionar qualquer automação de qualquer lugar, só enviando uma mensagem. E todas as notificações das automações agendadas chegam no mesmo lugar: o chat do seu bot.

Como criar um bot no Telegram do zero

Passo a passo: criando seu bot
Abra o Telegram e pesquise por @BotFather na barra de busca.
Inicie uma conversa com o BotFather e envie o comando /newbot.
O BotFather vai pedir um nome para o bot (pode ser qualquer nome, como "Meu Assistente").
Em seguida, vai pedir um username para o bot. Deve terminar em "bot" (exemplo: meuassistente_bot).
O BotFather vai retornar um token no formato 123456789:ABCdefGHIjklMNOpqrSTUvwxYZ. Copie e guarde esse token.
Abra uma conversa com o seu novo bot e envie qualquer mensagem para ativar o chat.
Para descobrir o seu Chat ID, acesse: api.telegram.org/botSEU_TOKEN/getUpdates. O campo "chat":{"id": ...} é o seu Chat ID.

Como instalar e configurar o MCP do Telegram

Com o token em mãos, adicione o MCP do Telegram ao seu arquivo .mcp.json conforme mostrado no Capítulo 2. Depois, abra o Claude Code e confirme que o MCP está ativo:

Instrução ao Claude Code 
Use o MCP do Telegram para me enviar uma mensagem de teste
no chat ID 123456789 com o texto "MCP do Telegram funcionando."

Como usar o Claude Code para enviar mensagens via Telegram

Uma vez que o MCP está configurado, enviar mensagens pelo Claude Code é direto. Você pode pedir mensagens simples, mensagens formatadas com negrito e listas, ou mensagens com dados calculados na hora.

Exemplos de instruções para envio de mensagens 
# Mensagem simples
Envie uma mensagem no meu Telegram dizendo que são 7h
e que o dia começou.

# Mensagem com dados do Calendar
Leia os eventos do meu Google Calendar para hoje
e envie um resumo formatado no Telegram com cada evento
em uma linha, no formato: HH:MM - Título do evento.

# Mensagem com cálculo do Sheets
Leia a planilha de gastos do Google Sheets, some todos
os gastos desta semana e envie o total no Telegram.

Como configurar o bot para receber mensagens e acionar o Claude Code

Para que o bot receba mensagens e acione automações, você precisa de um script que fique escutando as mensagens recebidas e chame o Claude Code quando uma mensagem chegar. O Claude Code pode criar esse script para você:

Instrução ao Claude Code 
Crie um script em Python que:
1. Fique escutando mensagens no meu bot do Telegram
   (token: SEU_TOKEN_AQUI)
2. Quando receber a mensagem "resumo do dia", execute
   o Claude Code com a instrução de ler os eventos do
   Google Calendar de hoje e responder no Telegram
3. Quando receber qualquer outra mensagem, responda
   "Comando não reconhecido."
O script deve rodar continuamente.

Exemplo concreto: bot que responde com a agenda do dia

Este é o fluxo completo do exemplo mais básico deste ebook: você envia "resumo do dia" para o seu bot, e ele responde com os eventos do Google Calendar de hoje.

O que acontece por dentro: o script Python recebe a mensagem, identifica o comando "resumo do dia", chama o Claude Code com a instrução de ler o Calendar, o Claude Code usa o MCP do Google Calendar para buscar os eventos, formata o resultado e usa o MCP do Telegram para enviar a resposta de volta para você. Tudo isso acontece em segundos.

Como testar se o bot está funcionando

Envie a mensagem "resumo do dia" para o seu bot. Se receber uma resposta com os eventos do dia, está funcionando. Se receber "Comando não reconhecido", o script está rodando mas não reconheceu o comando. Se não receber nada, o script pode não estar rodando.

Google Calendar: sua agenda inteligente

O Google Calendar guarda informações sobre o seu tempo. Com o MCP certo, o Claude Code consegue ler, criar e resumir esses dados sem que você precise abrir o aplicativo.

Como instalar e configurar o MCP do Google Calendar

O MCP do Google Calendar usa as credenciais do Google Cloud Console. O processo de configuração é feito uma única vez e vale também para o Gmail e o Google Sheets, já que todos usam a mesma conta Google.

Os passos são: acessar console.cloud.google.com, criar um novo projeto, ativar a API do Google Calendar, criar credenciais do tipo "OAuth 2.0 Client ID", baixar o arquivo JSON com as credenciais e usar o fluxo de autorização para gerar o refresh token. O Claude Code pode guiar você por cada um desses passos se você pedir:

Instrução ao Claude Code 
Me guie passo a passo para configurar as credenciais
do Google Calendar no Google Cloud Console e gerar
o refresh token necessário para o MCP.

O que o Claude Code consegue fazer com o Google Calendar

Capacidades do MCP do Google Calendar
Ler eventos: listar eventos de um dia, semana ou período específico, com horário, título, descrição e participantes.
Criar eventos: criar novos eventos com título, data, horário, duração, descrição e convidados.
Editar eventos: alterar horário, título ou descrição de eventos existentes.
Resumir agenda: gerar um texto formatado com os compromissos do dia ou da semana.

Exemplo concreto: resumo matinal automático no Telegram

Esta é uma das automações mais úteis do ebook: todo dia às 7h da manhã, o Claude Code lê os eventos do dia no Google Calendar e manda um resumo formatado no Telegram. Você acorda com a agenda já no celular, sem precisar abrir nenhum aplicativo.

Para criar essa automação, peça ao Claude Code:

Instrução ao Claude Code 
Crie uma automação agendada que:
1. Rode todo dia às 7h da manhã
2. Leia os eventos do meu Google Calendar para o dia atual
3. Formate a mensagem assim:
   "Bom dia! Hoje é [dia da semana], [data].
   Seus compromissos de hoje:
   [HH:MM] - [Título do evento]
   (repita para cada evento)
   Total: [N] compromisso(s)."
4. Se não houver eventos, envie: "Agenda livre hoje."
5. Envie a mensagem no meu Telegram (chat ID: SEU_ID)
Use cron job para o agendamento.

Como pedir ao Claude Code para criar eventos com linguagem natural

Uma das vantagens do Claude Code sobre ferramentas tradicionais é que você não precisa preencher formulários. Você descreve o evento em linguagem natural e o agente interpreta e cria.

Exemplos de criação de eventos por linguagem natural 
# Evento simples
Crie um evento no meu Google Calendar para amanhã
às 15h com o título "Reunião com dentista", duração
de 1 hora.

# Evento com detalhes
Crie um evento recorrente toda segunda-feira às 9h
com o título "Planejamento semanal", duração de 30
minutos, e adicione na descrição: "Revisar tarefas
da semana anterior e definir prioridades."

# Evento a partir de informação de e-mail
Leia o último e-mail do Gmail com assunto "Convite"
e crie um evento no Calendar com as informações
de data, horário e local que estiverem no e-mail.

Como combinar Google Calendar com Gmail

Uma das combinações mais práticas é criar eventos a partir de e-mails. Quando você recebe um e-mail com uma data de reunião, uma confirmação de consulta ou um convite de evento, o Claude Code pode ler o e-mail e criar o evento no Calendar automaticamente.

Instrução ao Claude Code 
Leia os últimos 10 e-mails não lidos do Gmail.
Para cada e-mail que contenha uma data e horário
de compromisso (reunião, consulta, evento, entrega),
crie um evento no Google Calendar com o título
extraído do e-mail, a data e horário encontrados,
e o nome do remetente na descrição.
Me informe no Telegram quais eventos foram criados.
Dica de uso

Rode essa instrução uma vez por semana para garantir que nenhum compromisso recebido por e-mail fique fora da agenda. O Claude Code é bom em identificar datas em diferentes formatos, inclusive escritas por extenso.

Gmail: sua caixa de entrada sob controle

E-mail é onde a maioria das pessoas perde mais tempo. Ler, classificar, responder: o Claude Code pode assumir boa parte desse trabalho.

Como instalar e configurar o MCP do Gmail

O MCP do Gmail usa as mesmas credenciais do Google Cloud Console que o Google Calendar. Se você já configurou o Calendar, o processo é quase idêntico: basta ativar também a API do Gmail no mesmo projeto do Google Cloud e garantir que o escopo de permissão inclua leitura e envio de e-mails.

Um detalhe importante: o Gmail exige que você especifique os escopos de acesso durante a autorização. Para as automações deste ebook, você vai precisar de permissão para ler e-mails, modificar labels e rascunhar respostas. Peça ao Claude Code para guiar a configuração:

Instrução ao Claude Code 
Configure o MCP do Gmail com permissões para:
- Ler e-mails (gmail.readonly)
- Modificar labels (gmail.modify)
- Criar rascunhos (gmail.compose)
Me guie pelo processo de autorização OAuth.

O que o Claude Code consegue fazer com o Gmail

Capacidades do MCP do Gmail
Ler e-mails: acessar e-mails por remetente, assunto, data ou label, incluindo o corpo completo da mensagem.
Classificar: aplicar labels, marcar como lido, mover para pastas ou arquivar e-mails com base em critérios definidos por você.
Resumir: gerar um texto com os pontos principais de um e-mail ou de um conjunto de e-mails.
Rascunhar respostas: criar um rascunho de resposta com base no conteúdo do e-mail original e nas suas instruções.

Exemplo concreto: resumo dos e-mails não lidos com prioridade

Esta automação lê todos os e-mails não lidos, classifica por prioridade e manda um resumo no Telegram. Você vê o que é urgente sem precisar abrir o Gmail.

Instrução ao Claude Code 
Leia todos os e-mails não lidos do Gmail.
Classifique cada um em três categorias:
- URGENTE: requer resposta hoje
- IMPORTANTE: requer atenção, mas não hoje
- INFORMATIVO: newsletters, notificações, avisos

Envie um resumo no Telegram neste formato:
"Caixa de entrada: [N] e-mails não lidos

URGENTE ([N]):
- [Remetente]: [Assunto resumido]

IMPORTANTE ([N]):
- [Remetente]: [Assunto resumido]

INFORMATIVO ([N] e-mails, sem detalhar)"

Como criar filtros inteligentes com linguagem natural

Filtros no Gmail tradicional exigem que você configure regras por remetente, assunto ou palavras-chave. Com o Claude Code, você descreve o comportamento que quer e ele aplica as labels e ações correspondentes.

Instrução ao Claude Code

Como construir automações complexas passo a passo

A abordagem mais eficiente para automações com múltiplos MCPs é construir em camadas. Primeiro, crie e teste a parte de leitura de dados. Depois, adicione o processamento. Por último, adicione o envio do resultado.

Se algo der errado em uma automação complexa, você vai saber exatamente em qual camada o problema está, porque cada uma foi testada separadamente antes de ser combinada.

Como testar automações complexas antes de deixar rodando

Antes de agendar uma automação para rodar sozinha, execute-a manualmente pelo terminal algumas vezes com dados reais. Verifique se o resultado no Telegram está formatado como esperado, se os dados no Sheets ou Notion foram registrados corretamente e se o comportamento quando não há dados (agenda vazia, sem e-mails) está correto.

Checklist antes de agendar

Execute manualmente três vezes: uma com dados normais, uma com dados ausentes (agenda vazia, sem e-mails) e uma com dados em excesso (muitos eventos, muitos e-mails). Se as três funcionarem, a automação está pronta para ser agendada.

Rodando automações na nuvem

Automações agendadas só funcionam se o computador estiver ligado. Para rodar de madrugada ou quando você estiver fora, a nuvem resolve.

Por que algumas automações precisam rodar na nuvem

Automações acionadas pelo Telegram ou pelo terminal funcionam bem no computador local: você está presente, o computador está ligado. Mas automações agendadas para rodar às 7h da manhã, ou às 22h de domingo, dependem de o computador estar ligado naquele momento.

A solução é hospedar a automação em um servidor na nuvem, que fica ligado 24 horas por dia. O servidor executa o script no horário programado, independente do estado do seu computador.

A plataforma recomendada: Railway

O Railway é uma plataforma de hospedagem que tem um plano gratuito sem necessidade de cartão de crédito, interface simples e suporte a deploy direto do GitHub. Para automações pessoais leves, o plano gratuito é suficiente.

Outras opções gratuitas existem (Render, Fly.io, Google Cloud Run), mas o Railway tem a curva de aprendizado mais baixa para quem não tem experiência com infraestrutura.

Como criar uma conta no Railway

Passo a passo
Acesse railway.app e clique em "Start a New Project".
Faça login com sua conta do GitHub (necessário para o deploy).
Confirme o e-mail se solicitado.
Você está pronto para criar o primeiro projeto.

Como pedir ao Claude Code para preparar a automação para o Railway

O Claude Code pode preparar todos os arquivos necessários para o deploy. Você só precisa descrever o que a automação faz e pedir a preparação:

Instrução ao Claude Code 
Prepare a automação de resumo matinal para deploy
no Railway. A automação roda todo dia às 7h, lê

5. Um README.md explicando como fazer o deploy

Quando algo der errado

Automações quebram. Saber diagnosticar o problema é tão importante quanto saber criar a automação.

Os erros mais comuns em automações com MCP

Erros frequentes e o que significam
Erro de autenticação (401 Unauthorized): o token ou a chave de API expirou ou está incorreto. Solução: regenerar as credenciais e atualizar o .mcp.json.
MCP não encontrado: o servidor MCP não iniciou. Verifique se o nome no .mcp.json está correto e se o pacote está instalado.
Permissão negada (403 Forbidden): as credenciais estão corretas, mas não têm permissão para a ação solicitada. Verifique os escopos de autorização.
Timeout: a operação demorou demais. Pode ser instabilidade na API da ferramenta. Tente novamente em alguns minutos.
Dados não encontrados: o ID da planilha, base de dados ou chat está errado. Verifique os IDs no CLAUDE.md.

Como descrever um erro ao Claude Code para que ele corrija

Quando uma automação falha, copie a mensagem de erro completa e cole no Claude Code junto com uma descrição do que a automação deveria fazer. Quanto mais contexto você der, mais rápido o agente identifica o problema.

Modelo de descrição de erro 
A automação de resumo matinal parou de funcionar.
O erro que aparece é:
[cole aqui a mensagem de erro completa]

O que a automação deveria fazer:
- Rodar todo dia às 7h
- Ler o Google Calendar
- Enviar mensagem no Telegram

O que aconteceu antes de parar:
- Funcionou normalmente por 2 semanas
- Parou ontem sem mudanças no código

Por favor, identifique o problema e corrija.

O que fazer quando um MCP perde a autenticação

Tokens do Google expiram periodicamente. O refresh token dura mais, mas também pode ser revogado se você alterar a senha da conta ou revogar o acesso manualmente no painel do Google. Quando isso acontece, o MCP retorna erro 401.

A solução é refazer o fluxo de autorização OAuth para gerar um novo refresh token. Peça ao Claude Code:

Instrução ao Claude Code 
O MCP do Google Calendar está retornando erro 401.
Me guie pelo processo de renovar o refresh token
usando as credenciais que já tenho no .mcp.json
(Client ID e Client Secret).

Para o Telegram, o token do bot não expira automaticamente, mas pode ser revogado se você pedir ao BotFather. Se isso acontecer, gere um novo token com o comando /token no BotFather e atualize o .mcp.json.

Como testar uma automação isoladamente

Quando uma automação com múltiplos MCPs falha, o primeiro passo é isolar qual parte está com problema. Teste cada MCP separadamente antes de testar a automação completa:

Instrução ao Claude Code 
# Teste isolado do Google Calendar
Use apenas o MCP do Google Calendar e liste
os eventos de hoje. Não use nenhum outro MCP.

# Teste isolado do Telegram
Use apenas o MCP do Telegram e envie a mensagem
"teste" no chat ID SEU_ID. Não use nenhum outro MCP.

Se cada MCP funcionar isoladamente, o problema está na lógica de combinação. Se um MCP falhar isoladamente, o problema está nas credenciais ou configuração daquele MCP específico.

Como desativar uma automação que está se comportando de forma inesperada

Se uma automação agendada está enviando mensagens erradas, registrando dados incorretos ou executando ações não esperadas, a forma mais rápida de parar é remover ou comentar o cron job que a aciona. No Railway, você pode pausar o serviço diretamente pelo painel sem precisar deletar o projeto.

Regra de segurança

Nunca deixe uma automação que acessa Gmail ou cria eventos no Calendar rodando sem ter testado o comportamento com dados ausentes ou inesperados. Um loop mal configurado pode criar dezenas de eventos ou enviar dezenas de mensagens antes de você perceber.

O que vem a seguir

Você chegou ao final deste ebook com um conjunto concreto de habilidades. Sabe como configurar MCPs para cinco ferramentas diferentes, como descrever automações ao Claude Code de forma que ele entenda, como combinar múltiplas ferramentas em uma única rotina e como hospedar automações na nuvem para que rodem sem depender do seu computador.

Mais do que as automações específicas que você aprendeu, o que importa é o padrão de raciocínio: identificar uma tarefa repetitiva, definir o gatilho e a ação, descrever com precisão ao Claude Code e testar antes de deixar rodando. Esse padrão funciona para qualquer automação que você queira criar a partir de agora.

Como pensar nas próximas automações

O melhor ponto de partida para a próxima automação é a sua própria rotina. Observe durante uma semana quais tarefas você faz de forma mecânica, sem pensar. Essas são as candidatas. Depois, aplique o filtro: é repetitiva, previsível e chata? Se sim, vale automatizar.

Não tente criar a automação perfeita de primeira. Comece com a versão mais simples que resolve o problema. Depois de ela estar funcionando, adicione complexidade gradualmente: mais fontes de dados, mais ações, mais condições. Automações evoluem com o uso.

O ecossistema de MCPs está crescendo

Quando este ebook foi escrito, os MCPs cobertos aqui representavam uma fração do que está disponível. Novos servidores MCP surgem toda semana: para Slack, Spotify, GitHub, Trello, Linear, Airtable, WhatsApp Business, sistemas de CRM e dezenas de outras ferramentas.

O repositório oficial de MCPs em modelcontextprotocol.io lista os servidores disponíveis e mantidos pela comunidade. Vale visitar periodicamente para descobrir novas integrações que podem se encaixar na sua rotina.

O próximo passo natural

O próximo passo natural é combinar as automações individuais que você criou em rotinas completas. A rotina matinal que você montou no Capítulo 8 é um exemplo: ela não é uma automação, é uma orquestração de várias automações rodando em sequência.

À medida que você acumula automações funcionando, começa a ver conexões entre elas. O registro de gastos no Sheets alimenta o relatório semanal. As tarefas do Notion aparecem no resumo matinal. Os e-mails do Gmail viram eventos no Calendar. Cada automação nova se conecta às que já existem, e o sistema como um todo fica mais útil do que a soma das partes.

Esse é o ponto onde a automação pessoal deixa de ser uma coleção de scripts e passa a ser uma infraestrutura de produtividade que trabalha por você enquanto você faz outras coisas.

"A melhor automação é aquela que você esquece que existe, porque simplesmente funciona."
