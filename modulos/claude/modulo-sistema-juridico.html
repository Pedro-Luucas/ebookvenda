O que você vai construir

Este tutorial existe para uma finalidade específica: ao final dele, você terá um sistema de gestão de processos jurídicos funcionando no seu computador, criado por você, sem escrever uma linha de código.

O sistema que você vai construir é uma aplicação web que roda localmente, ou seja, no seu próprio computador. Você acessa pelo navegador, como qualquer site, mas não precisa de internet, não precisa de servidor externo e não precisa pagar por nenhuma infraestrutura. Tudo fica na sua máquina.

O sistema gerencia processos jurídicos. Cada processo tem número, nome do cliente, tipo de ação (cível, trabalhista, penal e outros), tribunal e vara, status atual, última movimentação registrada, próxima ação prevista, data do prazo e advogado responsável. Você pode criar, editar, visualizar e excluir processos. Pode buscar por cliente, filtrar por status e por responsável, e ordenar por prazo.

O ponto central do sistema é o controle de prazos. Processos com prazo próximo recebem alertas visuais automáticos em três níveis: sete dias antes, três dias antes e um dia antes. Esses alertas aparecem na interface, com cores e ícones distintos para cada nível de urgência. O dashboard exibe um painel com o total de processos ativos, um gráfico de processos por status e a lista dos próximos prazos ordenada por urgência.

Para quem serve este sistema

O sistema foi desenhado para escritórios de advocacia de pequeno e médio porte, advogados autônomos e assistentes jurídicos que precisam de uma ferramenta simples para acompanhar processos sem depender de sistemas caros ou complexos. Também serve para qualquer pessoa que queira criar uma ferramenta de gestão para uso próprio e queira aprender como fazer isso com o Claude Code.

Não é necessário saber programar. O único pré-requisito é ter o Claude Code instalado e saber o básico de como dar instruções a ele: abrir o terminal, navegar até uma pasta e digitar comandos simples.

O que está fora do escopo

Este tutorial não cobre autenticação de usuários com senha e controle de acesso por perfil. O sistema não tem login. Qualquer pessoa com acesso ao computador pode usar. Se você precisar de controle de acesso, isso seria uma evolução futura.

O sistema não envia notificações por e-mail ou SMS. Os alertas de prazo aparecem apenas na interface, enquanto o sistema está aberto no navegador. Não há integração com calendários externos, como Google Calendar ou Outlook, embora isso seja tecnicamente possível como extensão futura.

O sistema não é publicado na internet. Ele roda apenas no computador onde foi instalado. Não há deploy em servidor, não há domínio, não há acesso remoto. Se você precisar acessar de outro computador ou de um celular, isso exigiria uma configuração adicional que está fora do escopo deste tutorial.

Evoluções como relatórios em PDF, integração com sistemas de tribunais, histórico de movimentações com upload de documentos e módulo financeiro são possibilidades reais, mas não fazem parte deste tutorial. O foco aqui é construir uma base sólida e funcional.

Preparando o projeto

Antes de escrever qualquer instrução ao Claude Code, você precisa preparar o ambiente. Isso significa criar a pasta do projeto, iniciar o Claude Code dentro dela e escrever o arquivo que vai guiar todo o desenvolvimento.

Criando a pasta do projeto

Escolha um local no seu computador onde você guarda projetos ou documentos importantes. Pode ser a área de trabalho, a pasta Documentos ou qualquer outro lugar de fácil acesso. Crie uma nova pasta e dê a ela um nome simples, sem espaços e sem acentos. Um bom nome seria sistema-juridico.

No Windows, você pode criar a pasta pelo Explorador de Arquivos: clique com o botão direito em um espaço vazio, selecione "Novo" e depois "Pasta". No macOS, o processo é o mesmo pelo Finder.

Depois de criar a pasta, abra o terminal. No Windows, você pode usar o Prompt de Comando, o PowerShell ou o Terminal do Windows. No macOS, use o Terminal. Navegue até a pasta que você criou usando o comando cd. Por exemplo:

Terminal 
cd C:\Users\SeuNome\Desktop\sistema-juridico
# No macOS:
cd ~/Desktop/sistema-juridico

Iniciando o Claude Code dentro da pasta

Com o terminal aberto na pasta do projeto, inicie o Claude Code com o comando:

Terminal 
claude

O Claude Code vai iniciar e mostrar a interface de linha de comando. A partir daqui, todas as instruções que você der serão executadas no contexto da pasta sistema-juridico. O Claude Code vai criar arquivos, instalar dependências e rodar comandos sempre dentro dessa pasta.

O CLAUDE.md: o passo mais importante do projeto

O arquivo CLAUDE.md é um documento de texto que fica na raiz da pasta do projeto. Ele serve como um briefing permanente para o Claude Code: toda vez que o agente inicia uma sessão, ele lê esse arquivo e entende o contexto do projeto, as regras que deve seguir e as decisões que já foram tomadas.

Pense no CLAUDE.md como as instruções que você daria a um desenvolvedor que acabou de entrar no projeto. Sem esse documento, o Claude Code começa cada sessão sem memória do que foi feito antes. Com ele, o agente sabe exatamente o que está construindo, qual tecnologia usar e quais convenções respeitar.

Um CLAUDE.md bem escrito tem quatro partes: a descrição do sistema (o que é e para que serve), a stack técnica (quais tecnologias usar), as convenções do projeto (como nomear arquivos, onde colocar cada coisa) e o escopo (o que está dentro e o que está fora). Quanto mais claro e específico for esse documento, menos retrabalho você vai ter.

# Sistema de Gestão de Processos Jurídicos

## Descrição
Aplicação web local para gestão de processos jurídicos em escritórios
de advocacia. Roda no computador do usuário via localhost. Sem deploy
externo, sem autenticação, sem envio de e-mail ou SMS.

## Stack técnica
- Framework: Next.js com App Router e TypeScript
- Estilo: Tailwind CSS
- Estado global: Zustand
- Banco de dados: SQLite (arquivo local, via better-sqlite3)
- API: rotas no próprio Next.js (app/api/)
- Porta padrão: 3000

## Estrutura de pastas
- app/          : páginas e rotas de API (Next.js App Router)
- components/   : componentes React reutilizáveis
- store/        : stores do Zustand
- types/        : tipos TypeScript compartilhados
- lib/          : utilitários e conexão com o banco de dados

## Banco de dados
Arquivo: juridico.db na raiz do projeto.
Tabelas: processos, advogados.
Nunca usar ORM. Usar better-sqlite3 diretamente com queries SQL.

## Convenções
- Português brasileiro em toda a interface visível ao usuário
- Nomes de arquivos em kebab-case (ex: lista-processos.tsx)
- Componentes React em PascalCase (ex: ListaProcessos)
- Datas sempre no formato DD/MM/AAAA na interface
- Datas armazenadas no banco como texto ISO 8601 (AAAA-MM-DD)

## Funcionalidades do sistema
1. CRUD completo de processos
2. Dashboard com totais, gráfico de status e próximos prazos
3. Sistema de alertas de prazo: 7 dias (amarelo), 3 dias (laranja),
   1 dia (vermelho), vencido (cinza escuro)
4. Busca por nome do cliente
5. Filtro por status (em andamento, concluído, suspenso)
6. Filtro por advogado responsável
7. Ordenação por data de prazo

## Campos de cada processo
- id (gerado automaticamente)
- numero_processo (texto)
- nome_cliente (texto)
- tipo_acao (cível, trabalhista, penal, família, outro)
- tribunal (texto)
- vara (texto)
- status (em_andamento, concluido, suspenso)
- ultima_movimentacao (texto)
- proxima_acao (texto)
- data_prazo (texto ISO 8601)
- advogado_responsavel (texto)
- criado_em e atualizado_em (texto ISO 8601, automáticos)

## Fora do escopo
- Autenticação e controle de acesso
- Envio de e-mail ou SMS
- Deploy em servidor externo
- Upload de documentos
- Integração com sistemas de tribunais

Skills que você vai precisar

O Claude Code tem um sistema de skills: módulos especializados que você instala e aciona para dar ao agente capacidades específicas. Para este projeto, cinco skills fazem diferença real.

Por que skills importam neste projeto

Sem skills, o Claude Code funciona como um agente generalista. Ele consegue criar código, corrigir erros e responder perguntas, mas sem a profundidade de um especialista em cada área. As skills mudam isso: quando você aciona a skill frontend-design, por exemplo, o agente passa a aplicar princípios de design visual que normalmente exigiriam anos de experiência em desenvolvimento de interfaces.

Para um sistema jurídico, onde a interface precisa ser clara e profissional e onde erros de lógica no sistema de prazos podem ter consequências sérias, usar as skills certas no momento certo é a diferença entre um sistema que funciona bem e um que funciona mais ou menos.

As cinco skills deste projeto

frontend-design

Esta skill especializa o Claude Code em design de interfaces. Ela aplica princípios de hierarquia visual, espaçamento, tipografia e uso de cores para gerar interfaces que parecem ter sido feitas por um designer profissional. Para o sistema jurídico, ela vai garantir que o dashboard seja legível, que os alertas de prazo sejam visualmente distintos e que o formulário de cadastro seja intuitivo.

Use esta skill sempre que for pedir ao Claude Code para criar ou ajustar qualquer tela do sistema.

react-nextjs-development

Esta skill especializa o agente no desenvolvimento com React e Next.js, especificamente com o App Router, que é a arquitetura que este projeto usa. Ela garante que os componentes sejam criados da forma correta para o Next.js moderno: sabe quando usar Server Components (componentes que rodam no servidor) e quando usar Client Components (componentes que rodam no navegador), e aplica as convenções corretas de nomenclatura e organização de arquivos.

Use esta skill ao criar páginas, componentes e rotas de API.

testing-debugging

Esta skill especializa o agente em identificar e corrigir erros. Quando algo não funciona como esperado, acionar esta skill antes de descrever o problema faz com que o Claude Code aborde o diagnóstico de forma mais sistemática: ele vai verificar os logs, identificar a causa raiz e propor uma correção precisa em vez de tentar soluções aleatórias.

Use esta skill sempre que algo não funcionar corretamente.

simplify

Esta skill revisa o código gerado em busca de complexidade desnecessária. Ao longo do desenvolvimento, é comum que o código acumule soluções redundantes, funções duplicadas ou estruturas mais complicadas do que o necessário. A skill simplify identifica esses problemas e os corrige, mantendo o projeto limpo e fácil de manter.

Use esta skill ao final de cada capítulo, depois de implementar uma funcionalidade completa.

typescript-expert

Esta skill especializa o agente no uso correto do TypeScript. O TypeScript é uma versão do JavaScript com tipagem estática, o que significa que ele verifica se os dados têm o formato correto antes de o sistema rodar. Sem essa skill, o Claude Code pode gerar código TypeScript que funciona mas que não aproveita os benefícios da tipagem, deixando brechas para erros difíceis de detectar.

Use esta skill ao criar os tipos compartilhados do projeto e ao revisar as interfaces de dados.

Uma nota sobre MCP

O Claude Code tem um sistema chamado MCP (Model Context Protocol), que permite conectar o agente a ferramentas externas. Por exemplo, existe um MCP para o Google Calendar que permitiria sincronizar os prazos do sistema jurídico diretamente com a agenda do Google. Existe também um MCP para o Notion, para o GitHub e para diversas outras ferramentas.

Essas integrações são tecnicamente possíveis e podem ser muito úteis em uma evolução futura do sistema. No entanto, elas estão fora do escopo deste tutorial. O foco aqui é construir o sistema base de forma sólida. Integrações externas adicionam complexidade e dependências que tornariam este tutorial mais longo e mais difícil de seguir para quem está começando.

Dica: não acione skills desnecessariamente

Acionar uma skill para uma tarefa simples não causa problemas, mas pode tornar as respostas do Claude Code mais longas do que o necessário. Reserve as skills para as tarefas em que elas fazem diferença real: design visual, estrutura de componentes, diagnóstico de erros e revisão de tipos.

O que é o SQLite em linguagem simples

O SQLite é um banco de dados que funciona como um arquivo no seu computador. Diferente de bancos de dados como MySQL ou PostgreSQL, que precisam de um servidor rodando em segundo plano, o SQLite é apenas um arquivo com extensão .db. Quando o sistema precisa salvar ou buscar dados, ele abre esse arquivo, faz a operação e fecha.

Para este projeto, o arquivo se chama juridico.db e fica na raiz da pasta do projeto. Todos os processos cadastrados ficam dentro desse arquivo. Para fazer backup do sistema inteiro, basta copiar esse arquivo para outro lugar.

Configurando o estado global com Zustand

Com o banco de dados configurado, o próximo passo é criar o estado global do sistema. O estado global é o mecanismo que permite que diferentes telas do sistema compartilhem e atualizem os mesmos dados sem precisar buscá-los do banco toda vez.

O que é estado global em linguagem simples

Imagine que você tem três telas no sistema: o dashboard, a lista de processos e o formulário de edição. Quando você edita um processo no formulário e salva, o dashboard e a lista precisam mostrar os dados atualizados. Sem estado global, cada tela buscaria os dados do banco de forma independente, o que causaria inconsistências e lentidão.

O estado global resolve isso criando um repositório central de dados que todas as telas podem ler e atualizar. Quando uma tela atualiza um processo, todas as outras telas que exibem aquele processo são atualizadas automaticamente, sem precisar recarregar a página.

Pense no estado global como uma lousa na sala de reuniões do escritório. Qualquer pessoa pode ler o que está escrito e qualquer pessoa pode apagar e reescrever. Quando alguém atualiza a lousa, todos que estão olhando para ela veem a mudança imediatamente.

O que é o Zustand

O Zustand (pronuncia-se "tsushtand", palavra alemã para "estado") é uma biblioteca para gerenciar estado global em aplicações React. Ele foi escolhido para este projeto por três razões práticas: é simples de configurar, tem uma API direta sem muita cerimônia e funciona bem com o Next.js App Router.

Alternativas como Redux ou Context API do React também funcionariam, mas o Zustand exige menos código para fazer a mesma coisa. Para um projeto criado com Claude Code por alguém sem experiência em programação, menos código significa menos pontos de falha e menos complexidade para o agente gerenciar.

Como pedir ao Claude Code para criar a store de processos

A store (em português, "armazém") é o arquivo que define o estado global e as ações que podem modificá-lo. Acione a skill typescript-expert antes de dar esta instrução:

Criando as rotas de API

As rotas de API são a ponte entre a interface que o usuário vê e o banco de dados onde os dados ficam armazenados. Toda vez que você cria, edita ou busca um processo, é uma rota de API que recebe o pedido, faz a operação no banco e devolve o resultado.

O que são rotas de API em linguagem simples

Imagine que o banco de dados é um arquivo em um cofre. A interface do sistema é a recepção do escritório. As rotas de API são os funcionários que ficam entre a recepção e o cofre: quando a recepção precisa de um documento, ela pede ao funcionário, que vai ao cofre, pega o documento e traz de volta.

No Next.js com App Router, as rotas de API são arquivos chamados route.ts dentro da pasta app/api/. Cada arquivo define o que acontece quando o sistema recebe um tipo específico de pedido: GET (buscar dados), POST (criar dados), PUT (atualizar dados) ou DELETE (excluir dados).

As rotas que o sistema precisa

Rotas necessárias e o que cada uma faz
GET /api/processos: retorna a lista de todos os processos. Aceita parâmetros opcionais de busca (cliente), filtro (status, advogado) e ordenação (prazo).
POST /api/processos: recebe os dados de um novo processo e o salva no banco. Retorna o processo criado com o id gerado.
GET /api/processos/[id]: retorna os dados de um processo específico pelo seu id.
PUT /api/processos/[id]: recebe os dados atualizados de um processo e os salva no banco. Retorna o processo atualizado.
DELETE /api/processos/[id]: remove um processo do banco pelo seu id.
GET /api/advogados: retorna a lista de advogados cadastrados para popular o campo de seleção no formulário.

O sistema de prazos

O sistema de prazos é o coração desta aplicação. Em um escritório de advocacia, perder um prazo processual pode ter consequências graves: recursos não interpostos, contestações não apresentadas, prescrições consumadas. Este capítulo trata essa funcionalidade com a seriedade que ela merece.

Por que o sistema de prazos é a parte mais crítica

Diferente de outras funcionalidades do sistema, como a busca ou o formulário de cadastro, o sistema de prazos precisa ser correto em todos os casos possíveis. Um erro no formulário de cadastro é inconveniente. Um erro no sistema de prazos pode fazer com que um alerta não apareça quando deveria, e isso tem consequências reais.

Por isso, este capítulo vai além de simplesmente pedir ao Claude Code para implementar a funcionalidade. Ele explica a lógica em detalhes, mostra como verificar cada caso e descreve os erros mais comuns para que você saiba o que procurar.

Como funciona a lógica de alertas

A lógica de alertas é baseada na diferença entre a data atual e a data do prazo de cada processo. Essa diferença, medida em dias, determina o nível de urgência do alerta:

Os cinco níveis de alerta
Mais de 7 dias: sem alerta. O prazo está confortável. Nenhuma cor especial ou ícone de urgência.
7 dias ou menos (e mais de 3): alerta amarelo. O prazo está se aproximando. Badge amarelo com ícone de relógio.
3 dias ou menos (e mais de 0): alerta laranja. O prazo está próximo. Badge laranja com ícone de aviso.
Prazo hoje (0 dias de diferença): alerta vermelho. Badge vermelho com ícone de urgência.
Prazo vencido (data no passado): alerta cinza escuro. O prazo já passou. Badge cinza com ícone de prazo expirado.

Um detalhe importante: o cálculo correto da diferença em dias precisa ignorar o horário. Dois processos com prazo no mesmo dia devem ter o mesmo nível de alerta, independentemente de um ter sido cadastrado às 9h e outro às 17h. A comparação deve ser feita apenas com as datas, sem considerar horas, minutos ou segundos.

Como pedir ao Claude Code para implementar a lógica

A lógica de prazos deve ser implementada como uma função utilitária separada, não embutida diretamente nos componentes. Isso facilita os testes e garante que a mesma lógica seja usada em todos os lugares do sistema.

1. Um tipo PrazoUrgencia:
   'normal' | 'atencao' | 'urgente' | 'critico' | 'vencido'

Erros comuns nessa etapa e como descrevê-los

Erro: fuso horário incorreto

O JavaScript pode interpretar datas ISO 8601 sem horário como meia-noite UTC, fazendo com que uma data apareça como o dia anterior dependendo do fuso horário. Se os alertas estiverem sempre um dia adiantados ou atrasados, descreva: "Os alertas de prazo estão sempre um dia errados. Suspeito de problema com fuso horário no cálculo de datas."

Erro: badge não atualiza sem recarregar

Se o badge não atualizar quando você edita um processo, o componente pode não estar reagindo às mudanças na store. Descreva: "Quando edito a data de prazo de um processo e volto para a lista, o badge ainda mostra a data antiga. Preciso recarregar a página para ver o badge atualizado."

Quando algo der errado

Erros fazem parte do desenvolvimento de software. Não são sinal de que algo está fundamentalmente errado: são parte normal do processo. O que importa é saber como lidar com eles de forma eficiente.

Os tipos de erro mais comuns em projetos Next.js feitos com Claude Code

Projetos criados com Claude Code por pessoas sem experiência em programação tendem a ter três categorias de erro:

A primeira categoria são erros de ambiente: o Node.js não está instalado, a versão está desatualizada, um pacote não foi instalado corretamente. Esses erros geralmente aparecem logo no início, quando você tenta iniciar o servidor pela primeira vez. A mensagem de erro costuma mencionar o nome do pacote ou módulo que está faltando.

A segunda categoria são erros de lógica: o sistema faz algo diferente do que deveria. Um processo é salvo com a data errada, a lista não aparece, um botão não responde. Esses erros são mais sutis e às vezes só aparecem durante o uso real do sistema.

A terceira categoria são erros de integração: o frontend e o backend não estão se comunicando corretamente. A tela tenta buscar dados de uma rota que não existe, ou envia dados em um formato que o servidor não entende. Esses erros geralmente aparecem no console do navegador (acessível pressionando F12) ou no terminal onde o servidor está rodando.

Como descrever um erro ao Claude Code de forma que ele consiga corrigir

A qualidade da correção depende diretamente da qualidade da descrição do problema. Um erro bem descrito tem quatro elementos:

Os quatro elementos de um bom relato de erro
1. O que você fez: descreva a sequência de ações que levou ao erro. "Abri o sistema, fui para a tela de novo processo, preenchi todos os campos e cliquei em Salvar."
2. O que esperava: descreva o resultado esperado. "Esperava que o processo aparecesse na lista de processos."
3. O que aconteceu: descreva o resultado real. "Em vez disso, a tela ficou em branco e não aconteceu nada."
4. A mensagem de erro: copie e cole qualquer mensagem de erro que aparecer, seja no terminal, seja no console do navegador (F12). Não resuma: copie o texto completo.

Quando o Claude Code não consegue corrigir sozinho

Em alguns casos, o Claude Code vai tentar corrigir um erro e a correção vai criar outro erro. Isso pode acontecer quando o problema é mais profundo do que parece na superfície, ou quando a instrução original foi ambígua e o agente tomou uma decisão que não era a ideal.

Se você perceber que o agente está tentando a mesma abordagem repetidamente sem sucesso, ou que cada correção cria um novo problema, pare. Não continue pedindo correções incrementais. Em vez disso, descreva o problema do início: o que o sistema deveria fazer, o que está fazendo de errado e o que já foi tentado para corrigir. Às vezes, recomeçar a descrição do problema do zero leva o agente a uma abordagem completamente diferente que resolve o problema de forma mais limpa.

Se o problema persistir após duas ou três tentativas com abordagens diferentes, considere desfazer as últimas mudanças e voltar ao estado anterior usando o Git, conforme explicado na próxima seção.

O que o sistema não faz

O sistema não tem autenticação. Qualquer pessoa com acesso ao computador pode usar. Não há controle de acesso por perfil de usuário. Não há envio de notificações por e-mail ou SMS: os alertas aparecem apenas na interface, enquanto o sistema está aberto no navegador. O sistema não está publicado na internet e não pode ser acessado de outros dispositivos sem configuração adicional.

Essas limitações são intencionais. Elas mantêm o sistema simples, fácil de manter e adequado para o escopo deste tutorial.

Possibilidades de evolução futura

O sistema que você construiu é uma base sólida. A partir dele, você pode pedir ao Claude Code para adicionar funcionalidades mais avançadas conforme a necessidade do escritório. Algumas possibilidades naturais: autenticação com login e senha, upload de documentos vinculados a cada processo, histórico de movimentações com registro de datas e responsáveis, relatórios em PDF, integração com o Google Calendar para sincronizar prazos com a agenda, e uma versão acessível de outros dispositivos na mesma rede local.

Cada uma dessas evoluções pode ser pedida ao Claude Code de forma incremental, sem precisar reescrever o sistema do zero. A estrutura que você criou, com banco de dados SQLite, rotas de API no Next.js e estado global com Zustand, suporta essas adições sem grandes mudanças arquiteturais.

Como continuar aprendendo Claude Code

Este tutorial mostrou um fluxo de trabalho específico: CLAUDE.md detalhado, skills acionadas no momento certo, instruções precisas e verificação a cada etapa. Esse fluxo funciona para qualquer tipo de sistema, não apenas para sistemas jurídicos.

Para projetos mais complexos, o mesmo princípio se aplica: quanto mais claro for o CLAUDE.md, quanto mais específicas forem as instruções e quanto mais frequentes forem os commits, melhor será o resultado. O Claude Code é uma ferramenta que amplifica a clareza das suas instruções: instruções vagas produzem resultados vagos, instruções precisas produzem resultados precisos.

A documentação oficial do Claude Code em docs.anthropic.com é o melhor lugar para aprender sobre funcionalidades avançadas: hooks, permissões, configurações de projeto e integrações via MCP. Com a base que você construiu neste tutorial, você tem o contexto necessário para aproveitar essas funcionalidades de forma produtiva.
