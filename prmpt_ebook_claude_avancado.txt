Sobre o ebook

Título: Claude Code Avançado: Automação, Agentes e Controle Total

Objetivo declarado no ebook: Este ebook é um guia técnico e prático para desenvolvedores que já dominam o uso básico do Claude Code e querem ir além: automatizar fluxos de trabalho com hooks, criar servidores MCP customizados, orquestrar múltiplos agentes, integrar o Claude Code em pipelines de CI/CD, usar o SDK programaticamente e configurar o ambiente com precisão cirúrgica. O foco é produtividade real em projetos reais.

Público-alvo: Desenvolvedores que sabem programar, já usam Claude Code, e querem dominar os recursos avançados para automatizar, escalar e integrar a ferramenta em seus fluxos de trabalho profissionais.

Tom: Técnico, direto e denso. Sem explicações básicas. O leitor sabe o que é um hook, sabe o que é CI/CD, sabe o que é TypeScript. Escreva como se estivesse documentando para um colega sênior. Mostre código real, configurações reais, casos de uso reais.

Extensão: Entre 45 e 60 páginas, proporcional à complexidade técnica de cada capítulo.

Formato: Ebook completo com texto corrido, capítulos bem desenvolvidos, subtítulos, blocos de código extensos e precisos, tabelas de referência onde couber.

Stack e ferramentas abordadas

- Claude Code CLI (@anthropic-ai/claude-code)
- Anthropic SDK (@anthropic-ai/sdk) para uso programático
- MCP SDK (@modelcontextprotocol/sdk) para servidores customizados
- GitHub Actions com claude-code-action
- Git worktrees para isolamento de agentes
- settings.json / settings.local.json / .mcp.json
- Hooks: PreToolUse, PostToolUse, Stop, Notification
- Custom slash commands em .claude/commands/
- CLAUDE.md hierárquico para monorepos

Regras de escrita

1. Nunca use "—" (travessão). Substitua por vírgula, dois pontos, ponto e vírgula ou reescreva.
2. Escreva em português brasileiro correto e natural.
3. Termos técnicos em inglês sem boa tradução podem ser usados sem explicação (o leitor é desenvolvedor).
4. Tom técnico e direto. Sem introduções longas, sem repetição de conceitos básicos.
5. Não use palavras como "incrível", "revolucionário", "poderoso". Mostre os fatos e o código.
6. Cada capítulo deve ter subtítulos internos.
7. Blocos de código devem ser completos, funcionais e comentados onde necessário.
8. Não resuma. Escreva o texto completo conforme os limites de palavras indicados.
9. O ebook deve fluir como documentação técnica de alta qualidade, não como tutorial para iniciantes.
10. Priorize exemplos reais e casos de uso concretos sobre explicações abstratas.
11. Tabelas de referência são bem-vindas para campos de configuração, variáveis de ambiente e flags de CLI.
12. O capítulo 3 (hooks) e o capítulo 5 (MCP) são os mais importantes. Trate-os com profundidade máxima.
13. Cada bloco de código deve ser autocontido: o leitor deve conseguir copiar e usar sem contexto adicional.
